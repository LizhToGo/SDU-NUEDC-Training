# TB6612 驱动使用说明

本文档对应当前工程里的 `main.syscfg`、`BSP/inc/bsp_tb6612.h` 和 `BSP/src/bsp_tb6612.c`。

## 1. 当前接线

| 模块 | 信号 | 工程 IO | 用途 |
|---|---|---|---|
| TB6612 | PWMA | PA0 / A00 | 右轮 PWM |
| TB6612 | PWMB | PA1 / A01 | 左轮 PWM |
| TB6612 | AIN1 | PB6 / B06 | A 路方向 1，实车右轮 |
| TB6612 | AIN2 | PB7 / B07 | A 路方向 2，实车右轮 |
| TB6612 | BIN1 | PB8 / B08 | B 路方向 1，实车左轮 |
| TB6612 | BIN2 | PB9 / B09 | B 路方向 2，实车左轮 |
| TB6612 | STBY | PB1 / B01 | 驱动使能，高电平有效，初始化默认低 |

当前 SysConfig 中 `PWMA` 使用 `TIMA0_CCP0`，`PWMB` 使用 `TIMA0_CCP1`。方向脚和 `STBY` 都属于 `GPIOB`。

## 2. 初始化写在哪里

TB6612 的初始化分为两层：外设初始化和驱动逻辑初始化。

### 2.1 引脚配置写在 main.syscfg

`main.syscfg` 是引脚配置的源头。你在这里指定：

```text
PWMA -> PA0 / TIMA0_CCP0
PWMB -> PA1 / TIMA0_CCP1
AIN1 -> PB6
AIN2 -> PB7
BIN1 -> PB8
BIN2 -> PB9
STBY -> PB1
```

如果以后换线，优先改 `main.syscfg`，不要直接改 `Debug/ti_msp_dl_config.c`，因为 `Debug` 里的文件是自动生成的。

### 2.2 实际 GPIO/PWM 初始化在 Debug/ti_msp_dl_config.c

CCS 构建时会根据 `main.syscfg` 自动生成：

```text
Debug/ti_msp_dl_config.h
Debug/ti_msp_dl_config.c
```

其中 `SYSCFG_DL_init()` 会调用：

```c
SYSCFG_DL_GPIO_init();
SYSCFG_DL_PWM_0_init();
```

`SYSCFG_DL_GPIO_init()` 负责把 `PA0/PA1` 配成 PWM 复用输出，把 `PB6/PB7/PB8/PB9/PB1` 配成普通 GPIO 输出。

`SYSCFG_DL_PWM_0_init()` 负责初始化 `TIMA0`，设置 PWM 周期、输出模式、比较通道等。

### 2.3 main.c 里调用初始化

程序入口里需要先初始化芯片外设，再初始化 TB6612：

```c
SYSCFG_DL_init();
TB6612_Init();
```

`SYSCFG_DL_init()` 来自 TI SysConfig，负责初始化时钟、GPIO、PWM、UART 等外设。

`TB6612_Init()` 写在 `BSP/src/bsp_tb6612.c`，它会先保持 `STBY` 为低，再设置刹车状态并拉高 `STBY`，避免上电后乱转。

## 3. 头文件里的宏有什么用

`BSP/inc/bsp_tb6612.h` 中的这些宏用于控制 TB6612 方向脚和使能脚：

```c
AIN1_OUT(0);
AIN2_OUT(1);
BIN1_OUT(0);
BIN2_OUT(1);
STBY_OUT(1);
```

这些宏底层调用的是 TI DriverLib：

```c
DL_GPIO_setPins(...);
DL_GPIO_clearPins(...);
```

也就是说，`AIN1_OUT(1)` 就是把 `AIN1` 对应的 GPIO 拉高，`AIN1_OUT(0)` 就是拉低。

## 4. 速度范围

驱动使用 `-999 ~ 999` 表示速度：

| 数值 | 含义 |
|---|---|
| 正数 | 正转 |
| 0 | PWM 为 0 |
| 负数 | 反转 |
| 999 | 当前配置下的最大 PWM 命令 |

如果输入超过范围，驱动会自动限幅到 `999`。

## 5. 常用函数

### TB6612_Init()

初始化 TB6612 驱动逻辑。

```c
TB6612_Init();
```

它内部会调用：

```c
TB6612_Enable();
TB6612_Brake();
```

所以初始化后，`STBY` 是高电平，电机处于刹车状态。

### TB6612_SetMotor()

控制单个电机。

```c
TB6612_SetMotor(TB6612_MOTOR_B, 300);   // 左轮正转
TB6612_SetMotor(TB6612_MOTOR_A, 300);   // 右轮正转
TB6612_SetMotor(TB6612_MOTOR_B, -200);  // 左轮反转
```

当前实车自检约定：`TB6612_MOTOR_A` 是右轮，`TB6612_MOTOR_B` 是左轮。

### TB6612_SetDifferential()

控制左右轮差速，后面做循迹控制主要用这个函数。

```c
TB6612_SetDifferential(250, 250);  // 直行
TB6612_SetDifferential(320, 150);  // 右转：左轮快，右轮慢
TB6612_SetDifferential(150, 320);  // 左转：左轮慢，右轮快
TB6612_SetDifferential(0, 0);      // PWM 输出为 0
```

### TB6612_Brake()

主动刹车。测试电机或调 PID 时建议用它停车。

```c
TB6612_Brake();
```

### TB6612_Coast()

滑行停止。它会关闭 PWM，同时把方向脚置为低电平，车会靠惯性滑一段。

```c
TB6612_Coast();
```

### TB6612_Enable() / TB6612_Disable()

控制 `STBY` 引脚。

```c
TB6612_Enable();   // STBY=1，允许驱动输出
TB6612_Disable();  // STBY=0，驱动待机
```

正常控制电机时不用反复手动调用 `TB6612_Enable()`，因为 `TB6612_Init()`、`TB6612_SetMotor()`、`TB6612_Brake()` 会自动使能。

## 6. 推荐接口

旧例程兼容接口 `AO_Control()`、`BO_Control()` 和 `TB6612_Motor_Stop()` 已删除。后续写循迹控制时，推荐优先用 `TB6612_SetDifferential()`，因为它直接表达“左轮速度、右轮速度”，更适合小车。

## 7. 方向校准

如果调用：

```c
TB6612_SetDifferential(250, 250);
```

小车不是向前走，有两种处理方式：

1. 交换对应电机的两根输出线。
2. 修改 `BSP/inc/bsp_tb6612.h` 中的方向宏：

```c
#define TB6612_A_FORWARD_IN1 (0U)
#define TB6612_A_FORWARD_IN2 (1U)
#define TB6612_B_FORWARD_IN1 (0U)
#define TB6612_B_FORWARD_IN2 (1U)
```

A 路对应右轮，B 路对应左轮。只反一个轮子时，只改对应那一组。

## 8. 推荐测试流程

先把车架空，避免突然冲出去，然后运行 `main.c` 里的 demo：

```c
TB6612_SetDifferential(250, 250);
TB6612_Brake();
TB6612_SetDifferential(300, 140);
TB6612_Brake();
TB6612_SetDifferential(140, 300);
TB6612_Brake();
```

确认左轮、右轮、前进方向都对之后，再接八路红外循迹模块和陀螺仪。
