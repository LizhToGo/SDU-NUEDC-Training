# JY62 陀螺仪驱动说明

本文档记录当前工程中 JY62/WT61 陀螺仪的接线、协议解析、软件接口和任务日志含义。当前 JY62 已经参与正式任务，不再作为单独的 JY62 UART 测试模式使用。

对应文件：

- `BSP/inc/bsp_jy62.h`：JY62 串口驱动，当前以头文件内 `static` 实现方式集成。
- `main.syscfg`：`UART_1` 引脚和波特率配置。
- `main.c`：启动置零、任务开始置零、直线航向修正、弧线航向进度判断和 JY62 日志打印。
- `app_config.h`：`ENABLE_JY62_NAV`、置零延时、打印周期和各任务航向修正参数。
- `docs/硬件接线说明.md`：JY62 实物接线表。

## 1. 当前接线

| JY62 信号 | MSPM0G3507 | SysConfig 名称 | 说明 |
|---|---|---|---|
| `RXD` | `PA08 / A08` | `UART_1_TX` | MCU 发送，接 JY62 接收脚 |
| `TXD` | `PA09 / A09` | `UART_1_RX` | MCU 接收，接 JY62 发送脚 |
| `VCC` | 按模块规格接 `3V3` 或 `5V` | - | 以手头模块供电要求为准 |
| `GND` | `GND` | - | 必须与主控、电机驱动、红外模块共地 |

注意：MCU 的 TX 接模块 RX，MCU 的 RX 接模块 TX。JY62 独占 `PA08/PA09` 的 UART1，调试输出和蓝牙串口仍走 `PA28/PA31` 的 UART0。

## 2. 串口参数

当前工程固定使用：

```text
115200 波特率，8 数据位，无校验，1 停止位
```

你们已经用维特上位机验证这块 JY62 在 `115200` 下通信正常。之前按部分商家 STM32 例程尝试 `9600` 时只能看到异常字节，因此后续不要随意切回 `9600`。

如果 `rx` 增长但 `frames` 不增长，优先检查：

- `PA08` 是否接 JY62 `RXD`。
- `PA09` 是否接 JY62 `TXD`。
- JY62、MSPM0、电机驱动和红外模块是否可靠共地。
- 模块当前波特率是否确实为 `115200`。

## 3. 数据帧格式

JY62 普通串口协议每帧 11 字节：

```text
byte0  byte1  byte2..byte9        byte10
0x55   类型    4 个 int16 数据      校验和
```

当前驱动解析三类帧：

| 类型 | 含义 | 数据 |
|---|---|---|
| `0x51` | 加速度帧 | `ax ay az temp` |
| `0x52` | 角速度帧 | `gx gy gz voltage` |
| `0x53` | 角度帧 | `roll pitch yaw version/invalid` |

每个数据都是小端 `int16`。校验和为前 10 个字节累加后取低 8 位，校验失败会累计到 `checksum_error`。

## 4. 换算单位

驱动内部使用整数定点换算，避免在 Cortex-M0+ 上引入浮点开销。

| 数据 | 驱动单位 | 换算 |
|---|---|---|
| 加速度 | `acc_mg`，毫 g | `raw / 32768 * 16g * 1000` |
| 角速度 | `gyro_mdps`，0.001 度/秒 | `raw / 32768 * 2000deg/s * 1000` |
| 角度 | `angle_cdeg`，0.01 度 | `raw / 32768 * 180deg * 100` |

当前任务控制主要使用 `yaw_relative_cdeg` 和 `gyro_z_filtered_mdps`。

## 5. 软件接口

当前导航结构体为 `jy62_navigation_t`，常用字段如下：

| 字段 | 含义 |
|---|---|
| `valid` | 已收到角速度帧和角度帧，并且数据可用于导航 |
| `update_flags` | 最近一次读取新增的数据类型，`0x07` 表示三类帧都更新过 |
| `yaw_cdeg` | 原始 yaw，单位 0.01 度 |
| `yaw_relative_cdeg` | 相对最近一次软件置零的 yaw，单位 0.01 度 |
| `gyro_z_mdps` | Z 轴角速度，单位 0.001 度/秒 |
| `gyro_z_filtered_mdps` | 低通后的 Z 轴角速度 |
| `rx_byte_count` | UART1 接收字节总数 |
| `header_count` | 识别到的 `0x55` 帧头数 |
| `frame_count` | 校验通过的有效帧总数 |
| `checksum_error / uart_error_count / overrun_count` | 校验错误、UART 硬件错误、FIFO 溢出计数 |

常用函数：

| 函数 | 用途 |
|---|---|
| `JY62_Init()` | 清空状态并开启 UART1 接收中断 |
| `JY62_SetYawZeroToCurrent()` | 把当前 yaw 记为软件零点 |
| `JY62_GetNavigation(&nav)` | 取导航数据并清掉本周期更新标志，适合打印状态 |
| `JY62_PeekNavigation(&nav)` | 只查看当前导航数据，不清更新标志，适合控制循环 |
| `JY62_UART1_IRQHandler()` | UART1 中断处理，必须由 `UART_1_INST_IRQHandler` 调用 |

## 6. 当前任务中的用法

工程通过 `app_config.h` 中的 `ENABLE_JY62_NAV (1)` 启用 JY62。

当前实际用法：

1. 上电后 `JY62_Init()`，等待 `JY62_BOOT_ZERO_DELAY_MS`，然后执行一次 `boot_zero`。
2. 空闲等待任务时，周期打印 `JY62 mode=idle ...`，用于确认模块仍在稳定输出。
3. 任务开始时重新把当前车头方向置零，减少人手摆车和启动姿态带来的误差。
4. 任务一和直线段使用 `yaw_relative_cdeg` 做航向修正，配合编码器速度 PID 和左右累计距离修正。
5. 任务三弧线段使用 yaw 变化量估计弧线进度，避免只靠编码器距离或红外瞬时状态误判出弯。
6. `gyro_z_filtered_mdps` 用于识别车身短时晃动，降低人手触碰或底盘抖动导致的误修正。

任务开始时如果 JY62 无效，会在日志里看到 `nav=0` 或 `zero=0`，这时优先排查 JY62 接线和波特率。

## 7. JY62 日志格式

典型日志：

```text
JY62 mode=TASK3_BD t=1000 df=213 ok=1 flags=0x07 yaw_cdeg=-12901 rel_cdeg=-11915 gz_mdps=-4577 gzlp_mdps=-305 rx=325071 head=29813 frames=29551 err=0/0/0
```

字段含义：

| 字段 | 含义 |
|---|---|
| `mode` | 打印场景，如 `boot_zero`、`idle`、`TASK1_AB`、`TASK3_BD` |
| `t` | 当前场景下的时间，单位 ms |
| `df` | 距离上次读取新增的有效帧数 |
| `ok` | 导航数据是否有效，正常应为 1 |
| `flags` | 最近更新的数据类型标志，常见 `0x07` |
| `yaw_cdeg` | 原始 yaw，单位 0.01 度 |
| `rel_cdeg` | 相对最近一次置零的 yaw，单位 0.01 度 |
| `gz_mdps` | Z 轴角速度，单位 0.001 度/秒 |
| `gzlp_mdps` | 低通后的 Z 轴角速度 |
| `rx` | UART1 接收字节总数 |
| `head` | 识别到的帧头总数 |
| `frames` | 校验通过的有效帧总数 |
| `err` | `校验错误/UART硬件错误/FIFO溢出` |

判断方法：

- `ok=1` 且 `frames` 持续增长：JY62 通信正常。
- `df` 约等于 150 帧/500 ms：115200 下输出频率正常。
- `err=0/0/0`：通信质量正常。
- `rel_cdeg` 在任务刚置零后应接近 0。
- 小车大幅转向时，`rel_cdeg` 应随车头方向连续变化。

## 8. 排错顺序

如果 JY62 数据异常，按下面顺序检查：

1. JY62 是否供电，GND 是否和主控、电机驱动、红外模块共地。
2. `PA08` 是否接 JY62 `RXD`，`PA09` 是否接 JY62 `TXD`。
3. JY62 波特率是否为 `115200`。
4. UART0 调试串口是否仍在看 `PA28/PA31`，不要把 JY62 和调试串口混接。
5. 如果 `rx/head` 增长但 `frames` 不增长，多半是波特率、线序或协议模式不对。
6. 如果 `overrun_count` 增长，说明 UART1 FIFO 曾经溢出；当前代码使用中断接收，正常运行时应保持 0。

调车时建议每次保存包含 `JY62 mode=...`、直线段 `h_raw/h_tgt/h_corr/gzlp`、弧线段 `yaw/ystep/yprog` 的日志片段，方便判断是航向零点、红外入线、还是编码器距离导致的偏差。
