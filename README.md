# 2026 电赛校赛 F 题：自动行驶小车

本仓库用于 2026 信息科学与工程学院电子设计竞赛选拔赛 F 题「自动行驶小车」的代码、资料整理和调试记录。项目目标是在 TI MSPM0 系列 MCU 控制下，让小车在指定双半圆弧赛道中自动行驶，完成过点声光提示、终点停车和多圈稳定运行。

本项目计划参考 [ZhijianLi2003/ZLC_MSPM0_Peripheral_Library](https://github.com/ZhijianLi2003/ZLC_MSPM0_Peripheral_Library) 的 MSPM0 小车方案，并根据本队硬件进行适配：

- 主控：立创·天猛星 MSPM0G3507 开发板
- 循迹：八路红外循迹模块
- 姿态：YJ62 陀螺仪
- 驱动：TB6612 电机驱动模块
- 底盘：万向轮 + 后驱差速

> 参考仓库使用 GPL-3.0 协议。若直接复制、修改并公开分发其代码，需要保留原协议与版权说明。

## 题目摘要

场地为白色哑光背景，面积不小于 `220cm x 120cm`。场地中只有两条对称半圆弧黑线，半径 `40cm`，线宽约 `1.8cm`，四个弧线顶点定义为 A、B、C、D。

小车约束：

- 尺寸不大于 `25cm x 15cm x 15cm`
- 采用轮式小车，不得使用履带和麦克纳姆轮
- 行驶过程中只能前进，不得后退
- 不得安装摄像头
- 采用车载电池供电，测试中途不得更换电池
- 经过 A/B/C/D 点和停车时必须有声光提示
- 启动、停车、经过 A/B/C/D 点时，小车地面投影必须覆盖圆弧顶点
- 圆弧行驶过程中，小车投影必须在弧线上，脱离弧线则该项测试失败

评分任务：

| 项目 | 路径要求 | 时间要求 | 分值 |
| --- | --- | --- | --- |
| 要求 1 | A -> B，B 点停车 | <= 15s | 20 |
| 要求 2 | A -> B -> C -> D -> A，A 点停车 | <= 30s | 20 |
| 要求 3 | A -> C -> B -> D -> A，A 点停车 | <= 40s | 30 |
| 要求 4 | 按要求 3 路径自动行驶 4 圈停车 | 越快越好 | 30 |
| 设计报告 | 系统方案、误差分析、电路与程序、测试数据 | - | 20 |

## 总体思路

这道题不是单纯巡线，而是“有黑线弧段 + 无标记直线段”的混合导航。场地除两条半圆弧外没有其他标记，因此：

- 直线段：依靠 YJ62 航向闭环 + 编码器里程闭环
- 弧线段：依靠八路红外循迹模块寻线 + YJ62/编码器判断出弧
- 过点提示：由路径状态机在每个运动段结束时触发声光提示

推荐拆成三个运动原语：

```text
straight_to(distance, heading)
follow_arc(direction, target_yaw_delta)
point_prompt(point)
```

再通过状态机组合路径：

```text
INIT -> CALIBRATE -> WAIT_START -> SEGMENT_x -> PROMPT_POINT -> STOP
```

## 硬件接口要点

### 八路红外循迹模块

模块支持三种读取方式：

| 方式 | 数据 | MSPM0G3507 示例接线 | 建议 |
| --- | --- | --- | --- |
| IO | 8 路数字量 | `PA28=x1, PA31=x2, PA02=x3, PB24=x4, PB20=x5, PB19=x6, PB18=x7, PA07=x8` | 最直观，但占 8 个 IO |
| I2C | 8 路数字量 | `PB3=SDA, PB2=SCL, 3V3, GND` | 当前暂行接线，推荐优先试，省 IO |
| USART | 数字量 + 模拟量 | `PA9=TX, PA8=RX, 5V, GND` | 能拿模拟量，但占串口 |

协议要点：

- I2C 地址：`0x12`
- I2C 数字量寄存器：`0x30`
- 串口启动数字量输出：`$0,0,1#`
- 串口启动模拟量输出：`$0,1,0#`
- 串口同时启动数字量和模拟量输出：`$0,1,1#`
- 数字量格式：`$D,x1:0,x2:0,...,x8:0#`
- 模拟量格式：`$A,x1:4096,x2:4096,...,x8:4096#`

校准流程：

1. 模块上电后等待约 `20s`。
2. 长按 `KEY1`，红灯常亮后进入校准模式。
3. 按实际安装高度把探头全部放在黑线上，静止 `3s`，轻按 `KEY1` 录入黑线值。
4. 把探头放在白底上，静止 `3s`，轻按 `KEY1` 录入白底值。
5. 红灯熄灭表示校准成功；慢闪表示失败，需要重新校准。

注意：示例代码中出现“白底为 1、黑线为 0”的描述，但正式控制前必须实测黑白逻辑。I2C 方式还要处理上电时序，建议做“上电延时 + I2C 超时恢复”。

### TB6612 电机驱动

关键参数：

- 电机驱动电压 `VM`：常见范围 `4.5V-13.5V`，稳压版模块资料写 `4.5V-15V`
- 逻辑电平 `VCC`：`2.7V-5.5V`
- PWM 频率：推荐 `10kHz`
- `STBY=1` 正常工作，`STBY=0` 待机
- 单路持续电流建议不超过 `1.2A`，峰值 `3.2A` 不可持续

简化真值表：

| PWM | IN1 | IN2 | 状态 |
| --- | --- | --- | --- |
| 1 | 1 | 0 | 正转 |
| 1 | 0 | 1 | 反转 |
| 1 | 0 | 0 | 制动停止 |
| 1 | 1 | 1 | 制动停止 |
| 0 | x | x | 停止 |

严禁电源反接、短路、带电插拔电机线、长时间堵转。

### YJ62 陀螺仪

当前资料来自维特 SDK，可优先按 UART Normal 协议移植：

- `WitInit(WIT_PROTOCOL_NORMAL, 0x50)`
- 串口收到的每个字节喂给 `WitSerialDataIn()`
- 用 `WitRegisterCallBack()` 注册寄存器更新回调
- 角度换算：`sReg[Roll/Pitch/Yaw] / 32768.0f * 180.0f`
- 示例会扫描 `4800` 到 `921600` 波特率，常见初始化波特率为 `9600`

天猛星 `JY61P` 移植包可作为 I2C 参考：

- I2C 地址：`0x50`
- 角度寄存器从 `0x3D` 开始，连续读 6 字节得到 Roll/Pitch/Yaw
- Yaw 寄存器：`0x3F`

YJ62 是否完全兼容 JY61P/维特寄存器协议，需要用实物确认。若 YJ62 是串口输出型，优先走 UART。

### 编码器

建议保留编码器。天猛星 N20 霍尔编码器示例提供了外部中断读取 A/B 相、定时器周期结算速度、`Motor_Get_Encoder(0/1)` 获取左右轮速度计数等逻辑。

注意：示例中的 `Motor_Set_PWM()` 右轮分支疑似把 `ABS(pwmb)` 误写成 `ABS(pwma)`，移植前必须修正并实测左右轮独立控制。

## 控制策略

### 直线段

直线段没有地面标记，建议使用：

- `yaw PID` 控制方向
- `speed PID` 控制左右轮速度
- 编码器积分控制距离
- 起步和刹车使用斜坡，减少打滑和万向轮摆动

### 弧线段

弧线段以红外循迹为主：

- 读取 8 路数字状态
- 统一黑白逻辑，得到 `hit_black[i]`
- 用加权法计算 `line_error`
- `line PID` 输出转向角速度
- 用 YJ62 累计角度或编码器弧长判断半圆结束

```text
weights = [-7, -5, -3, -1, 1, 3, 5, 7]
line_error = sum(hit_black[i] * weights[i]) / sum(hit_black[i])
```

若没有任何一路检测到黑线，应进入线丢失处理，根据最后一次偏差方向低速找线。

## 调试路线

1. 点亮最小系统：下载、串口打印、LED、蜂鸣器、按键。
2. 跑通底盘：确认左右轮方向、PWM 死区、最低可控速度、刹车距离。
3. 接入 YJ62：读取 yaw，完成清零、连续角处理和通信超时保护。
4. 接入八路红外循迹模块：确认黑白逻辑，完成校准，打印 8 路状态。
5. 完成直线航向保持：先跑 `50cm`、`100cm`，记录终点误差。
6. 完成单个半圆循迹：低速调 PID，确保不脱线。
7. 拼接要求 1：先完成 A -> B 停车，优先拿基础分。
8. 拼接要求 2/3：低速跑完整状态机，再逐步加速。
9. 挑战要求 4：要求 3 单圈稳定后，再做 4 圈稳定性测试。

## 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| 直线段无标记 | A/B/C/D 点定位误差大 | 编码器里程 + YJ62 航向闭环，多次标定 |
| 万向轮摆动 | 起步偏航、循迹抖动 | 降低起步加速度，优化重心，检查安装垂直度 |
| 红外循迹灵敏度漂移 | 弧线脱线 | 固定离地高度，赛前逐路确认黑白输出 |
| 陀螺仪漂移 | 多圈累计误差 | 启动前静止校准，弧线段用黑线修正轨迹 |
| 电池电压下降 | 速度和刹车距离变化 | 速度闭环，记录电压，预留时间裕量 |
| 车体超尺寸 | 失去测试资格 | 传感器、电池、线束装好后再测外廓 |
