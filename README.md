# SDU NUEDC Training：2026 校赛 F 题自动行驶小车

本仓库用于 2026 信息科学与工程学院电子设计竞赛选拔赛 F 题「自动行驶小车」的固件、接线文档和调试记录。当前工程基于 TI MSPM0G3507，使用 CCS + SysConfig + DriverLib 开发。

项目思路参考 [ZhijianLi2003/ZLC_MSPM0_Peripheral_Library](https://github.com/ZhijianLi2003/ZLC_MSPM0_Peripheral_Library)，但硬件已经按本队小车做了适配：

- 主控：立创·天猛星 MSPM0G3507 开发板
- 底盘：万向轮 + 后驱差速
- 电机驱动：TB6612 双路电机驱动模块
- 编码器：左右后轮霍尔编码器
- 循迹：八路红外循迹模块，当前使用 I2C 读取
- 姿态：YJ62 陀螺仪，待接入
- 串口：当前使用 UART0 打印调试信息，后续等待蓝牙串口模块到货后继续测试

## 当前程序模式

`main.c` 顶部有两个最常用的调试开关：

```c
#define ENABLE_IR_TRACKING_UART_TEST (1)
#define ENABLE_ENCODER_SELF_TEST (0)
```

### 八路红外循迹模块测试

当前默认：

```c
#define ENABLE_IR_TRACKING_UART_TEST (1)
```

程序会：

1. 初始化 SysConfig、编码器状态和 TB6612。
2. 让 TB6612 保持刹车，小车不会主动跑。
3. 通过 I2C 读取八路红外循迹模块。
4. 每 `100ms` 通过串口打印一次 `raw`、`mask`、`pos`、`err`。

串口输出示例：

```text
IR raw=0xE7 mask=0x18 count=2 lost=0 pos=0 err=0
```

字段含义：

| 字段 | 含义 |
|---|---|
| `raw` | 模块原始字节，`bit7=X1`，`bit0=X8` |
| `mask` | 程序转换后的黑线掩码，`bit0=X1`，`bit7=X8`，1 表示黑线 |
| `count` | 当前看到黑线的探头数量 |
| `lost` | 1 表示 8 路都没有看到黑线 |
| `pos` | 加权线位置，中心为 0，左负右正 |
| `err` | 循迹控制误差，当前等于 `pos` |

目前场地图实测结果正常：中间压线时 `mask=0x18`、`err=0`；黑线向左移动时误差为负；黑线向右移动时误差为正。

### 直行 PID 跑车测试

如果要恢复两轮直行 PID，把红外测试开关改成：

```c
#define ENABLE_IR_TRACKING_UART_TEST (0)
```

程序会进入 `run_motor_pid_stream()`，用编码器计数修正左右轮 PWM，使两轮速度尽量一致。

当前基础 PWM：

```c
#define STRAIGHT_B_BASE_PWM (300)   // 左轮 B 电机
#define STRAIGHT_A_BASE_PWM (310)   // 右轮 A 电机
```

PID 串口输出示例：

```text
PID t=1000 B_cnt=56 A_cnt=55 B_spd=56 A_spd=55 err=1 P=16 I=-65 D=0 corr=-2 B_pwm=302 A_pwm=308
```

其中 `err = B_spd - A_spd`：

- `err > 0`：左轮 B 比右轮 A 快，程序会降低 B 轮 PWM、提高 A 轮 PWM。
- `err < 0`：右轮 A 比左轮 B 快，程序会提高 B 轮 PWM、降低 A 轮 PWM。
- `err` 接近 0：左右轮速度比较一致。

## 当前接线摘要

完整接线请看 [暂行接线图.md](暂行接线图.md)。

### TB6612

| 信号 | MSPM0G3507 | 当前代码含义 |
|---|---|---|
| `PWMA` | `PA0 / TIMA0_CCP0` | A 电机 PWM，右轮 |
| `PWMB` | `PA1 / TIMA0_CCP1` | B 电机 PWM，左轮 |
| `AIN1` | `PB6` | A 电机方向 1 |
| `AIN2` | `PB7` | A 电机方向 2 |
| `BIN1` | `PB8` | B 电机方向 1 |
| `BIN2` | `PB9` | B 电机方向 2 |
| `STBY` | `PB1` | TB6612 使能 |

### 编码器

| 信号 | MSPM0G3507 |
|---|---|
| B 电机编码器 A 相 | `PA14` |
| B 电机编码器 B 相 | `PA15` |
| A 电机编码器 A 相 | `PA16` |
| A 电机编码器 B 相 | `PA17` |

### 八路红外循迹模块

| 信号 | MSPM0G3507 |
|---|---|
| `SDA` | `PB3 / I2C_0_SDA` |
| `SCL` | `PB2 / I2C_0_SCL` |
| `VCC` | 优先 `3V3` |
| `GND` | `GND` |

I2C 地址为 `0x12`，数字量寄存器为 `0x30`。

### 串口

| 信号 | MSPM0G3507 | 说明 |
|---|---|---|
| `UART0_TX` | `PA28` | MCU 发送，接串口模块 RX |
| `UART0_RX` | `PA31` | MCU 接收，接串口模块 TX |
| 波特率 | `115200` | 8 数据位，无校验，1 停止位 |

等待蓝牙串口模块到货后，优先用这一路做无线调试。注意不要让多个串口模块同时驱动 `PA31/RX`，否则可能互相冲突。

## 后续开发建议

下一步建议按这个顺序继续：

1. 蓝牙串口模块到货后，先只验证无线串口输出，不急着改控制逻辑。
2. 给 UART0 增加简单命令解析，例如启动、停车、切换打印频率、修改基础 PWM。
3. 接入 YJ62，先完成 yaw 读取、清零和连续角处理。
4. 用八路红外模块的 `err` 做低速循迹 PID，只跑单个半圆。
5. 把直线 PID、循迹 PID、YJ62 航向和编码器距离组合成路径状态机。
6. 最后再加入 A/B/C/D 过点声光提示和多圈停车逻辑。

## 协作约定

- `main` 分支尽量保持可编译、可烧录。
- 引脚、PWM、PID 参数发生变化时，同步更新 README 和对应文档。
- 不提交 `Debug/`、`Release/`、`.out`、`.map`、大型商家资料包、视频和安装包。
- 新增外设驱动时，优先放到 `BSP/inc` 和 `BSP/src`，并在 `docs/` 中补一份使用说明。
- 调试时先让小车架空，确认方向和串口输出正常后再落地跑。

## 赛题要点摘要

F 题要求小车在白色底板上的两条黑色半圆弧之间自动行驶，完成指定路径、过点声光提示和终点停车。约束包括：

- 小车尺寸不大于 `25cm x 15cm x 15cm`。
- 只能使用轮式小车，不能使用履带和麦克纳姆轮。
- 行驶过程中只能前进，不能后退。
- 不能安装摄像头。
- 圆弧行驶时，小车投影不能脱离黑线。
- 经过 A/B/C/D 点和停车时需要声光提示。

因此本项目后续控制策略会分成两类：

- 直线段：主要依靠编码器距离 + YJ62 航向闭环。
- 弧线段：主要依靠八路红外循迹模块 + YJ62/编码器判断出弧。
