# SDU NUEDC Training：2026 校赛 F 题自动行驶小车

本仓库用于 2026 信息科学与工程学院电子设计竞赛选拔赛 F 题「自动行驶小车」的固件、接线文档和调试记录。当前工程基于 TI MSPM0G3507，使用 CCS + SysConfig + DriverLib 开发。

项目思路参考 [ZhijianLi2003/ZLC_MSPM0_Peripheral_Library](https://github.com/ZhijianLi2003/ZLC_MSPM0_Peripheral_Library)，但硬件和控制参数已经按本队小车做了适配。

## 当前交接状态

准备 push 的当前版本已经能通过串口运行 `01/02/05/00`：

- `01`：任务一 A -> B 直线。
- `02`：任务二 A -> B -> C -> D -> A。
- `05`：左右轮编码器/PWM 测试。
- `00`：强制停车。

这一版直线段目前跑得还可以，已知主要问题是进入 B -> C / D -> A 弧线的一瞬间容易冲出赛道。当前没有恢复旧的循迹强制转向，也还没有合并其他 branch 里的数学前馈逻辑。后续调试重点和当前参数记录在 [docs/当前调试状态与后续问题.md](docs/当前调试状态与后续问题.md)。

## 当前稳定版进度

| 模块 | 状态 | 说明 |
|---|---|---|
| CCS / SysConfig 工程 | 已建立 | `main.syscfg` 是外设和引脚配置源头 |
| TB6612 驱动 | 已跑通 | A 电机为右轮，B 电机为左轮 |
| 编码器读取 | 已跑通 | GPIOA 中断解码 A/B 相，任务开始时清零距离计数 |
| 八路红外循迹模块 | 已跑通 | 任务一用于识别 B 点黑线并刹车 |
| JY62 陀螺仪 | 已跑通 | 任务开始时置零，参与直线航向修正 |
| 任务一 A->B | 已实测稳定 | PB21 第一次按下后启动，编码器 + PID + JY62 直行，到 B 点黑线停车 |
| 声光提示 | 已接入 | 任务开始和任务结束各触发一次 |
| 完整路径状态机 | 待实现 | 后续继续做任务二、任务三和弧线循迹 |

任务一当前策略：

1. 手动把小车放在 A 点。
2. 按下 PB21 / KEY2 第一次。
3. 声光模块提示一次。
4. 短暂等待，JY62 当前航向置零，避免人手松开时的晃动直接进入控制。
5. 小车用编码器速度 PID、左右轮总距离修正、JY62 航向修正共同保持直线。
6. 距离达到 `TASK1_B_LINE_ARM_COUNT` 后，开始允许红外模块识别 B 点黑线停车。
7. 如果没有识别到黑线，到 `TASK1_FORCE_STOP_COUNT` 会强制刹车，避免继续冲到弧线外。
8. 刹车后声光模块再次提示。

## 当前程序入口

常用开关都在 `app_config.h`：

```c
#define ENABLE_CONTEST_TASKS         (1)
#define ENABLE_IR_TRACKING_UART_TEST (0)
#define ENABLE_LINE_FOLLOW_TEST      (0)
#define ENABLE_JY62_NAV              (1)
```

当前默认进入正式任务调度：

- 上电后打印 BOOT 信息。
- 初始化 JY62、编码器、TB6612。
- 串口打印 `TASK ready: press PB21/KEY2...`。
- 按 PB21 第一次执行任务一 A->B。
- 后续按键次数暂时只打印未实现提示，任务二之后还没写。

调试模式保留在 `app_debug_modes.h` 中：

- `ENABLE_IR_TRACKING_UART_TEST=1`：只读红外模块并打印，不跑车。
- `ENABLE_LINE_FOLLOW_TEST=1`：纯红外循迹调试。
- `ENABLE_CONTEST_TASKS=0` 且其他调试开关关闭：进入基础直行 PID 连续测试。

## 串口日志

协作调参时优先看 [docs/串口调试与跑车流程.md](docs/串口调试与跑车流程.md)，里面记录了：

- `JY62 ...` 行每个字段的含义。
- `TASK1 ...` 周期日志里编码器、红外、JY62、PID、PWM 字段的含义。
- `TASK1 stop ...` 如何判断是黑线停车、强制停车还是超时。
- 根据日志判断直线是否稳定、红外是否漏检、I2C 是否异常、编码器或电机线是否有问题。

快速判断：

- 正常任务一通常应以 `TASK1 stop: reason=line` 结束。
- `ir=1` 表示红外 I2C 读取成功，`ir=0` 优先查红外模块供电、共地、SDA/SCL。
- `arm=0` 时看到黑线会被忽略，因为还没到允许停车距离。
- `reason=force` 表示没在允许距离后识别到黑线，程序按编码器距离兜底刹车。
- `corr` 为最终左右轮修正量，长期贴近限幅说明参数或硬件状态需要检查。

## 协作约定

- `main` 分支尽量保持可编译、可烧录、可复现实测稳定状态。
- 引脚、PWM、PID、阈值参数发生变化时，同步更新 README 和对应文档。
- 不提交 `Debug/`、`Release/`、`.out`、`.map`、大型商家资料包、视频和安装包。
- 新增外设驱动时，优先放到 `BSP/inc` 和 `BSP/src`，并在 `docs/` 中补一份使用说明。
- 调试时先让小车架空，确认方向、编码器和串口输出正常后再落地跑。

## 后续开发建议

1. 以当前任务一稳定版作为留档基线。
2. 在新分支上做任务二，先复用任务一的直线段，再逐步加入红外循迹弧线段。
3. 弧线段优先使用八路红外模块闭环，JY62 和编码器用于判断进入/退出弧线。
4. 最后再组合 A/B/C/D 过点声光提示、多任务按键计数和完整停车逻辑。
