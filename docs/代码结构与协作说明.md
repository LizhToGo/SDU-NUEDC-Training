# 代码结构与协作说明

本文记录当前项目的代码边界，方便后续协作、调车和继续重构。

## 总体原则

- 当前工程仍按 CCS/SysConfig 生成的 `Debug/makefile` 构建。
- 不手动修改 `Debug/` 下的生成文件。
- 新拆出的控制模块暂时采用 header-only 方式，避免新增 `.c` 文件没有被 CCS 自动链接。
- 调车参数集中放在 `app_config.h`，不要把临时经验值散落在业务函数里。
- 任务三和任务四共用竞速跑圈逻辑，改动时要同时考虑单圈和四圈。

## main.c 现在负责什么

`main.c` 现在只保留少量顶层职责：

- 系统启动：`SYSCFG_DL_init()`、JY62、编码器、TB6612 初始化。
- 任务分发：等待按钮或 UART0 命令。
- 任务一入口：`run_task1_ab()`。
- 任务二入口：`run_task2_abcd()`。
- 任务六测试入口：`run_task6_ac_c_turn_test()`。
- 中断入口：`GROUP1_IRQHandler()`、`UART_1_INST_IRQHandler()`。

如果要修改具体控制策略，通常不要先改 `main.c`，而是进入对应模块。

## include 关系提示

一些模块看起来像普通头文件，但内部包含 `static` 实现。它们依赖 `main.c` 前面已经定义的私有 helper，所以 include 位置有意义：

- `heading_straight.h` 放在任务四调试钩子之后。
- `arc_segment.h` 放在 Task2 弧线对准 helper 之后。
- `race_laps.h` 放在任务一/二/六入口之后、`run_task_dispatcher()` 之前。

如果移动这些 include，可能会出现隐式声明、静态函数顺序或行为差异。

## 任务三/四控制流

入口：

```c
run_race_laps(1U);  // 任务三
run_race_laps(4U);  // 任务四
```

阶段顺序：

```text
AC -> CB -> BD -> DA
```

每段含义：

| 阶段 | 控制方式 | 点位 |
|---|---|---|
| AC | 目标航向直线，红外用于到 C 点检测 | `C_LINE` |
| CB | 左弧线红外循迹 + 差速控制 | `B_EXIT` |
| BD | 目标航向直线，红外用于到 D 点检测 | `D_LINE` |
| DA | 右弧线红外循迹 + 差速控制 | `A_EXIT` 或 `A_FINISH` |

任务三跑一圈；任务四跑四圈。前几圈到 A 点后会继续转向进入下一圈，最后一圈到 A 点后完成。

## 串口命令

| 命令 | 作用 |
|---|---|
| `00` | 停车 |
| `01` | 任务一 |
| `02` | 任务二 |
| `03` | 任务三 |
| `04` | 任务四 |
| `05` | PID 轮速测试 |
| `06` | A->C 快速转向测试 |
| `07` | PD 轮速测试 |
| `10` | AB 零角度标定和航向监控 |

命令解析在 `app_task_ids.c`。它同时兼容 ASCII 数字和单字节 HEX 命令。

## 日志层级

日志分三类：

- 普通串口日志：默认输出启动、任务开始、点位和停止原因。
- Task2 RAM 日志：由 `TASK2_RAM_LOG_ENABLE` 控制，代码在 `task2_ram_log.h`。
- Race RAM 日志：由 `TASK11_RAM_LOG_ENABLE` 控制，代码在 `race_log.h`。

当前 `TASK11_*` 命名是历史 Task11 采集阶段留下的配置名。虽然名字仍是 `TASK11_*`，但它们现在服务于任务三/四的 `race_*` 控制逻辑。

## 继续重构时的安全顺序

1. 先确认 `gmake -C Debug all` 通过。
2. 只移动代码时，不改 `app_config.h` 参数。
3. 新增 `.c` 前先确认 CCS 工程会编译它；否则继续用 header-only。
4. 每次拆一个模块后立刻构建。
5. 控制逻辑重构和调参分开提交，方便实车问题回溯。

## 调车时的建议记录

每次上车测试至少记录：

- 当前 Git commit 或分支。
- 是否改过 `app_config.h`。
- 使用任务编号。
- 起点摆放姿态。
- 串口中的 `start`、关键 `RACE_DATA`、`RACE_POINT_STATE`、`stop`。
- 实车现象，例如偏左、偏右、提前入弯、出弯角度不足、丢线位置。
