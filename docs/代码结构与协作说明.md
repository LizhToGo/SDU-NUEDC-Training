## 3. 模块职责

### 3.1 main.c（510 行）

**职责**：
- 系统启动：`SYSCFG_DL_init()`、JY62、编码器、TB6612 初始化
- 任务分发：等待按钮或 UART0 命令
- 任务一/二入口
- 中断入口：`GROUP1_IRQHandler()`、`UART_1_INST_IRQHandler()`

**修改场景**：极少修改

### 3.2 race/race_laps.h

**职责**：竞速跑圈主逻辑

**主要结构体**：
- `line_result_t` — 停止原因快照
- `race_phase_config_t` — 阶段配置
- `race_context_t` — 竞速运行时状态

**主要函数**：
- `run_race_laps()` — 跑圈主函数
- `race_configure_phase()` — 配置阶段参数
- `race_update_loop_state()` — 更新循环状态
- `race_compute_loop_control()` — 计算控制量
- `race_check_phase_point()` — 检查点位触发
- `race_execute_point_action()` — 执行点位动作
- `race_advance_segment()` — 阶段过渡

**修改场景**：改路径、点位、转向动作

## 4. 协作规范

### 4.1 调参规范

1. 调参只改 `app_config.h`
2. 每次只改一个参数
3. 记录修改前后的值
4. 修改后重新编译验证
5. 实车测试时观察串口日志

### 4.2 代码修改规范

1. 控制逻辑修改按模块分离
2. 每次修改后用 CCS 编译验证
3. 新增模块需确认 CCS 工程配置
4. 不要随意移动 include 位置

### 4.3 提交规范

1. 提交前确认日志开关配置
2. 不要提交 `Debug/`、`.out`、`.map`
3. 说明改动类型（参数/结构/文档）

## 5. include 关系

### 5.1 header-only 模块

以下模块在 `main.c` 中按顺序 include：

```c
#include "heading/heading_straight.h"
#include "race/task2_ram_log.h"
#include "turn/arc_segment.h"
#include "race/race_laps.h"
```

### 5.2 include 顺序

1. `heading/heading_straight.h` 放在任务四调试钩子之后
2. `turn/arc_segment.h` 放在 Task2 弧线对准 helper 之后
3. `race/race_laps.h` 放在任务一/二/六入口之后、`run_task_dispatcher()` 之前

### 5.3 注意事项

- 不要随意移动 include 位置
- 移动 include 可能导致隐式声明或行为差异
- 新增模块需确认 CCS 工程会编译它

## 6. 任务流程

### 6.1 任务三/四流程

```text
AC（直线）→ CB（弧线）→ BD（直线）→ DA（弧线）→ 下一圈
```

### 6.2 入口

```c
run_race_laps(1U);  // 任务三
run_race_laps(4U);  // 任务四
```

### 6.3 阶段说明

| 阶段 | 控制方式 | 点位 |
|------|----------|------|
| AC | 目标航向直线 | C_LINE |
| CB | 左弧线红外循迹 | B_EXIT |
| BD | 目标航向直线 | D_LINE |
| DA | 右弧线红外循迹 | A_EXIT 或 A_FINISH |

2. 只移动代码时，不改 `app_config.h` 参数

5. 控制逻辑重构和调参分开提交
