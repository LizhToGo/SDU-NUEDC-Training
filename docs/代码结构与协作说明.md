# 代码结构与协作说明

当前工程已经进入验收版精简阶段：只保留任务一、任务二、任务三、任务四。历史调试任务和测试头文件已从源码中删除。

## 顶层流程

```text
main()
  -> SYSCFG_DL_init()
  -> JY62_Init()
  -> encoder_init_runtime()
  -> TB6612_Init()
  -> run_task_dispatcher()
```

`run_task_dispatcher()` 是唯一运行入口，负责等待实体按键或 UART0 命令。

## 任务入口

| 任务 | 触发 | 入口 |
|---|---|---|
| 任务一 | A26 / `01` | `run_task1_ab()` |
| 任务二 | A24 / `02` | `run_task2_abcd()` |
| 任务三 | B24 / `03` | `run_race_laps(1U)` |
| 任务四 | A22 / `04` | `run_race_laps(4U)` |

运行中 UART0 `00` 会触发停车。其他历史调试编号不再解析。

## 调参原则

- 默认只改 `app_config.h`。
- 任务四高速调参优先改 `RACE_TASK4_*`。
- 没有明确要求时，不要改任务一/二/三稳定参数。
- 不要恢复 05/06/07/08/10 调试入口；验收版只保留 01..04。
- 不要提交 `Debug/`、`.out`、`.map`、日志 CSV、串口输出等本地生成物。
