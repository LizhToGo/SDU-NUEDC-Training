# JY62 陀螺仪驱动说明

本文档对应当前工程中的：

- `BSP/inc/bsp_jy62.h`
- `main.c` 中的 `run_jy62_uart_test()`
- `main.syscfg` 中的 `UART_1`
- `暂行接线图.md` 中的 JY62 接线表

当前目标是把 JY62/WT61 的串口数据整理成小车导航真正需要的量，并通过 UART0/蓝牙串口打印导航摘要。此测试模式不会驱动电机，TB6612 会保持刹车。

## 1. 当前接线

| JY62 信号 | MSPM0G3507 | SysConfig 名称 | 说明 |
|---|---|---|---|
| `RXD` | `PA08 / A08` | `UART_1_TX` | MCU 发送，接 JY62 接收脚 |
| `TXD` | `PA09 / A09` | `UART_1_RX` | MCU 接收，接 JY62 发送脚 |
| `VCC` | 按模块规格接 `3V3` 或 `5V` | - | 以手头模块供电要求为准 |
| `GND` | `GND` | - | 必须与主控共地 |

注意：MCU 的 TX 接模块 RX，MCU 的 RX 接模块 TX。JY62 使用 `PA08/PA09`，调试输出和蓝牙串口仍然走 `PA28/PA31` 的 UART0。

## 2. 串口参数

你已经用维特上位机验证当前 JY62 在 `115200` 波特率下通信正常，因此本工程固定使用 `115200`。

之前按部分商家 STM32 例程尝试过 `9600`，串口里只能看到重复异常字节，说明当前这块模块并不是 `9600` 输出，后续调试不要再切回 `9600`。

```text
115200 波特率，8 个数据位，无校验，1 个停止位
```

当前 `main.syscfg` 按 `115200` 配置 `UART_1`。如果 `115200` 下仍然没有 `0x55` 帧头，优先检查 JY62 `TXD` 是否接到了 `PA09/UART1_RX`，以及 GND 是否可靠共地。

## 3. 数据帧格式

JY62 普通串口协议每帧 11 字节：

```text
byte0  byte1  byte2..byte9        byte10
0x55   类型    4 个 int16 数据      校验和
```

当前驱动解析三类帧：

| 类型 | 含义 | 数据 |
|---|---|---|
| `0x51` | 加速度帧 | `ax ay az temp` |
| `0x52` | 角速度帧 | `gx gy gz voltage` |
| `0x53` | 角度帧 | `roll pitch yaw version/invalid` |

根据 `WT61协议.pdf`，`0x52` 帧的第 4 个数据是电压，`0x53` 帧的第 4 个数据是版本号或无效值。当前导航不使用这些辅助数据。

每个数据都是小端 `int16`，例如：

```text
value = low_byte + high_byte * 256
```

校验和为前 10 个字节累加后取低 8 位。校验失败会累计到 `chkerr`。

## 4. 换算单位

换算公式来自商家 STM32 示例程序：

| 数据 | 驱动打印单位 | 换算 |
|---|---|---|
| 加速度 | `acc_mg`，毫 g | `raw / 32768 * 16g * 1000` |
| 角速度 | `gyro_mdps`，0.001 度/秒 | `raw / 32768 * 2000deg/s * 1000` |
| 角度 | `angle_cdeg`，0.01 度 | `raw / 32768 * 180deg * 100` |

为了减少 Cortex-M0+ 上的浮点开销，当前驱动用整数定点数打印。`yaw` 和 `gz` 会额外用小数格式打印，便于直接观察航向角和 Z 轴角速度。

## 5. 导航实际使用的数据

对当前小车跑弧线来说，JY62/WT61 里真正需要长期进入控制算法的是两项：

| 数据 | 来自帧 | 用途 |
|---|---|---|
| `yaw` | `0x53` 角度帧的偏航角 Z | 判断车头相对起点转过了多少角度，可用于圆弧段航向闭环 |
| `gz` | `0x52` 角速度帧的 Z 轴角速度 | 判断车身正在以多快速度转向，可作为转向阻尼，减少左右抖动 |

`roll`、`pitch` 只用于确认模块安装姿态是否正常；`acc_x/y/z` 暂时不参与导航控制。原因是赛道巡线的主要问题是平面内转向，Z 轴航向和 Z 轴角速度最直接，受地面震动影响也比加速度小。

当前驱动提供了一个面向导航的结构体 `jy62_navigation_t`，主要字段如下：

| 字段 | 含义 |
|---|---|
| `yaw_cdeg` | 原始偏航角，单位 0.01 度 |
| `yaw_relative_cdeg` | 软件归零后的相对偏航角，单位 0.01 度 |
| `gyro_z_mdps` | Z 轴角速度，单位 0.001 度/秒 |
| `gyro_z_filtered_mdps` | 低通后的 Z 轴角速度，后续建议用于转向阻尼 |
| `valid` | 已经收到 `0x52` 和 `0x53` 时为 1 |

软件零点在 `JY62_Init()` 后约 300 ms 重新设置一次，也就是测试程序启动稳定后，当前车头方向会被当作 `rel=0.00`。

## 6. 当前测试模式

`main.c` 中当前默认打开：

```c
#define ENABLE_JY62_UART_TEST (1)
```

烧录后串口会看到类似输出：

```text
JY62 UART test start
UART1: PA08 TX -> JY62 RXD, PA09 RX <- JY62 TXD, baud=115200, 8N1
NAV t=500 df=159 flags=0x07 ok=1 yaw=-18.70 rel=0.00 gz=-0.244 gzlp=-0.122 frames=159 err=0/0/0
```

字段含义：

| 字段 | 含义 |
|---|---|
| `t` | 程序进入 JY62 测试后的毫秒数 |
| `df` | 本次打印周期新增的有效帧数，115200/100Hz 时约 150 帧/500 ms |
| `flags` | 本周期收到的新数据类型，`0x02` 角速度，`0x04` 角度，`0x07` 表示三类帧都收到 |
| `ok` | 导航数据是否有效，正常应为 1 |
| `yaw` | 原始航向角，单位度 |
| `rel` | 软件归零后的相对航向角，单位度，后续跑圆弧主要看它 |
| `gz` | Z 轴角速度，单位度/秒 |
| `gzlp` | 低通后的 Z 轴角速度，单位度/秒 |
| `frames` | 校验通过的有效帧总数 |
| `err` | `校验错误/UART硬件错误/FIFO溢出`，正常应为 `0/0/0` |

如果 `rx` 增长但 `frames` 不增长，程序会额外打印：

```text
JY62 raw_tail=...
```

它表示最近收到的原始字节。排查时重点看里面是否能看到 `55 51`、`55 52`、`55 53` 这样的帧头和类型组合。

## 7. 排错顺序

如果没有有效数据，建议按下面顺序检查：

1. JY62 是否供电，GND 是否和主控共地。
2. `PA08` 是否接 JY62 `RXD`，`PA09` 是否接 JY62 `TXD`。
3. JY62 波特率是否为 `115200`。
4. 蓝牙串口是否仍在看 UART0 的输出，也就是 `PA28/PA31`。
5. 如果 `irq` 和 `rx` 都增长，但 `frames` 不增长，多半是波特率错误、线序错误或收到的不是普通 `0x55` 协议输出。
6. 如果 `ovr` 增长，说明 UART1 FIFO 曾经溢出；当前代码已经改为中断接收，正常测试时它应该保持 0。

后续如果要把 JY62 融入弧线控制，可以优先使用 `yaw` 做航向角闭环，使用 `gz` 做转向阻尼，避免只靠八路红外数字量在圆弧上来回抖动。
