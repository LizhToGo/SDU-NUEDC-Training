# 串口调试与跑车流程

本文记录当前版本如何通过 UART0 控制任务、观察串口日志，并根据字段判断小车状态。当前调试串口为 UART0，`115200 8N1`。

## 1. 串口配置

| 项目 | 当前值 |
|---|---|
| 串口实例 | `UART_0` |
| 发送引脚 | `PA28 / UART0_TX` |
| 接收引脚 | `PA31 / UART0_RX` |
| 波特率 | `115200` |
| 数据格式 | 8 数据位，无校验，1 停止位 |

接线时交叉连接：

| MSPM0G3507 | 串口模块 |
|---|---|
| `PA28 / TX` | 模块 `RX` |
| `PA31 / RX` | 模块 `TX` |
| `GND` | 模块 `GND` |

如果同时接 DAPLink 串口和蓝牙串口，不要让两个外部 TX 同时直接驱动 MSPM0 的 `PA31/RX`。

## 2. UART0 命令

| 命令 | 作用 |
|---|---|
| `01` | 启动任务一 A -> B 直线 |
| `02` | 启动任务二 A -> B -> C -> D -> A |
| `03` | 启动任务三 A -> C -> B -> D -> A |
| `04` | 启动任务四，按任务三路线连续 4 圈 |
| `05` | 启动左右轮编码器/PWM 测试 |
| `00` | 强制停车 |

任务命令被接受后会打印：

```text
TASK UART command accepted: id=...
```

上电后会打印：

```text
TASK ready: A26/UART0 01=task1, A24/UART0 02=task2, B24/UART0 03=task3, A22/UART0 04=task4, UART0 05=PWM test, UART0 00=stop
```

## 3. 实体按键

| 按键引脚 | 任务 |
|---|---|
| `A26 / PA26` | 任务一 |
| `A24 / PA24` | 任务二 |
| `B24 / PB24` | 任务三 |
| `A22 / PA22` | 任务四 |

四个按键均按低电平按下处理。任务开始前会先松手消抖，再进入对应任务。

## 4. JY62 日志

典型输出：

```text
JY62 mode=TASK3_BD t=1000 df=213 ok=1 flags=0x07 yaw_cdeg=-12901 rel_cdeg=-11915 gz_mdps=-4577 gzlp_mdps=-305 rx=325071 head=29813 frames=29551 err=0/0/0
```

| 字段 | 含义 |
|---|---|
| `mode` | 当前场景，如 `boot_zero`、`TASK1_AB`、`TASK3_BD` |
| `t` | 当前场景下时间，单位 ms |
| `df` | 距离上次读取新增的有效帧数 |
| `ok` | 导航数据是否有效 |
| `flags` | 最近更新的数据类型标志，常见 `0x07` |
| `yaw_cdeg` | JY62 原始 yaw，单位 0.01 度 |
| `rel_cdeg` | 相对最近一次置零的 yaw，单位 0.01 度 |
| `gz_mdps` | Z 轴角速度，单位 mdps |
| `gzlp_mdps` | 低通后的 Z 轴角速度 |
| `rx/head/frames` | UART1 接收字节数、帧头数、有效帧数 |
| `err` | `校验错误/UART错误/溢出` 计数 |

判断：

- `ok=1` 且 `frames` 增长：JY62 正常。
- `err` 持续增长：优先查线序、波特率、供电和共地。
- 任务开始置零后 `rel_cdeg` 应接近 0。

## 5. 直线段日志

任务一、任务二 AB/CD、任务三 AC/BD、任务四各圈 AC/BD 都使用同一套直线段日志。示例：

```text
TASK3_BD t=500 dist=2753 arm=0 arm_cnt=7000 fast=0 find=0 centered=0 stopok=0 raw=0xFF mask=0x00 cnt=0 lost=1 ir=1 ir_err=0 nav=1 h_raw=-11887 h_tgt=-11892 h_flt=-415 h_use=5 h_gain=100 h_wob=0 gzlp=-8117 h_corr=6 hp=2 B_total=2474 A_total=3032 d_err=-558 d_corr=-34 B_spd=104 A_spd=102 v_tgt=0 v_err=2 P=44 I=4 D=-6 v_corr=2 bal=0 search=0 corr=6 base=532/543 B_pwm=526 A_pwm=549
```

### 5.1 基本字段

| 字段 | 含义 |
|---|---|
| `t` | 当前段开始后的时间，单位 ms |
| `dist` | 当前段距离估计，`(abs(B_total)+abs(A_total))/2` |
| `arm` | 是否允许红外到线触发 |
| `arm_cnt` | 当前段红外触发允许距离 |
| `fast` | 是否使用任务二 AB 快速修正参数 |
| `find` | 是否进入寻线/摆头辅助 |
| `centered` | 红外是否满足居中条件 |
| `stopok` | 当前红外采样是否满足停车/入线条件 |
| `raw/mask/cnt/lost` | 红外原始值、黑线掩码、触发探头数、是否丢线 |
| `ir` | I2C 读取是否成功 |
| `ir_err` | 红外线位置误差 |
| `nav` | JY62 数据是否有效 |

### 5.2 航向字段

| 字段 | 含义 |
|---|---|
| `h_raw` | 当前 JY62 相对航向 |
| `h_tgt` | 目标航向，任务一/AC 为 0，BD 为计算后的目标 |
| `h_flt` | 航向滤波中间值 |
| `h_use` | 实际用于控制的航向误差 |
| `h_gain` | 晃动降权后的航向增益百分比 |
| `h_wob` | 车身晃动标志 |
| `gzlp` | 低通 Z 轴角速度 |
| `h_corr` | 航向修正量 |
| `hp` | 航向优先级，`2` 表示只按目标航向控制 |

### 5.3 编码器和 PID 字段

| 字段 | 含义 |
|---|---|
| `B_total/A_total` | 左右轮当前段累计编码器计数 |
| `d_err` | `B_total - A_total` |
| `d_corr` | 累计距离修正 |
| `B_spd/A_spd` | 左右轮当前周期速度估计 |
| `v_tgt` | 速度差目标 |
| `v_err` | 速度 PID 误差 |
| `P/I/D` | 速度 PID 三项中间值 |
| `v_corr` | 速度 PID 输出 |
| `bal` | 编码器平衡修正 |

### 5.4 PWM 字段

| 字段 | 含义 |
|---|---|
| `search` | 寻线辅助修正量 |
| `corr` | 最终综合修正量 |
| `base` | 当前基础 PWM，格式 `B/A` |
| `B_pwm/A_pwm` | 最终左轮 B、右轮 A PWM |

`corr > 0` 时降低 B 轮、提高 A 轮；`corr < 0` 时提高 B 轮、降低 A 轮。

## 6. 直线段停止日志

示例：

```text
TASK3_BD stop: reason=force t=1520 dist=9758 arm=7000 force=9500 raw=0xFF mask=0x00 cnt=0 lost=1 ir=1 nav=1 rel_cdeg=-12077 B_total=9479 A_total=10037
```

| 字段 | 含义 |
|---|---|
| `reason=line` | 红外线触发到点 |
| `reason=force` | 编码器距离兜底停车 |
| `reason=timeout` | 最大运行时间超时 |
| `reason=uart_stop` | UART0 `00` 强制停止 |
| `rel_cdeg` | 停止瞬间相对航向 |
| `B_total/A_total` | 停止瞬间左右轮累计计数 |

## 7. 任务三弧线日志

任务三 CB/DA 使用专用弧线日志。示例：

```text
TASK3_CB t=1500 dist=6790 yaw=17090 ystep=186 yprog=17555 nav=1 ir=1 valid=1 wide=0 final=0 yrdy=1 drdy=0 frdy=0 brdy=1 bmark=1 xcnt=0 seen=1 lost_cnt=0 entry=0 mode=0 raw=0xCF mask=0x0C cnt=2 err=-1000 filt=-999 der=0 turn=-90 L=340 R=550
```

| 字段 | 含义 |
|---|---|
| `yaw` | 当前相对航向 |
| `ystep/yprog` | 本周期弧线航向增量 / 累计弧线进度 |
| `valid` | 红外是否有效看到线 |
| `wide` | 是否看到宽线或多探头大面积触发 |
| `final` | DA 最终 A 点出线判定 |
| `yrdy/drdy/frdy/brdy` | 角度、距离、最终出线、B 点方向条件 |
| `bmark` | B 点出线方向锁存 |
| `xcnt` | 出线确认计数 |
| `seen/lost_cnt` | 是否曾见线 / 连续丢线次数 |
| `entry` | 是否处于入弯强化区 |
| `mode` | 0 正常，1 入弯丢线，2 入弯见线，3 中后段丢线 |
| `err/filt/der` | 红外误差、滤波误差、差分项 |
| `turn/L/R` | 转向量和最终左右轮 PWM |

任务三更多说明见 [任务三调试说明.md](任务三调试说明.md)。

## 8. 任务四日志

任务四会给每一圈的四段加上圈数 tag：

```text
TASK4_L1_AC
TASK4_L1_CB
TASK4_L1_BD
TASK4_L1_DA
...
TASK4_L4_DA
```

前三圈 A 点出线后不会停车，会打印下一圈 AC 的目标角种子：

```text
TASK4_A target seed: lap=1 A_exit=... rel_turn=-6700 next_target=... mode=relative
```

含义：

| 字段 | 含义 |
|---|---|
| `A_exit` | 当前圈 DA 出 A 点时的 JY62 相对航向 |
| `rel_turn` | A 点换圈转角，当前 `-6700`，与 B 点 `+6700` 等大反向 |
| `next_target` | 下一圈 AC 直线段目标航向 |

第四圈 `TASK4_L4_DA` 到 A 点后会刹车并触发结束声光。

任务四更完整的换圈逻辑和调参建议见 [任务四调试说明.md](任务四调试说明.md)。

## 9. 05 轮速测试

UART0 发送 `05` 后进入轮速测试。典型输出：

```text
PID t=1000 B_cnt=... A_cnt=... B_total=... A_total=... dist_count=... d_err=... d_corr=... B_spd=... A_spd=... diff=... B_avg=... A_avg=... diff_avg=... target_diff=... err=... P=... I=... D=... ff=... fb=... corr=... B_pwm=... A_pwm=...
```

用途：

- 检查 A/B 电机是否都转。
- 检查左右编码器是否都计数。
- 标定 `COUNTS_PER_CM`。
- 判断某次突然转圈是否为电机线、编码器线或 TB6612 接线松动。

发送 `00` 可以退出 05 模式。

## 10. 常见问题

| 现象 | 优先检查 |
|---|---|
| 串口没有输出 | 波特率、TX/RX 交叉、串口占用、供电 |
| `ir=0` | 红外模块供电、共地、SDA/SCL、I2C 地址 |
| `nav=0` 或 JY62 `frames` 不增长 | JY62 供电、共地、TX/RX、波特率 |
| 一侧 `B_spd/A_spd` 长期为 0 | 电机线、编码器线、TB6612 方向脚 |
| `reason=force` | 到点线没扫到，检查角度、红外高度、arm 距离和寻线 |
| `corr` 长期贴近限幅 | 先查硬件，再调 PWM/PID/航向权重 |
| 起步突然大转向 | 检查电机线松动、JY62 是否被手碰到、起步置零是否稳定 |
