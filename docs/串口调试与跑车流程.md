# 串口调试与跑车流程

---

## 1. 串口命令

| 命令 | 作用 | 代码入口 |
|------|------|----------|
| `00` | 停车 | `TASK_ID_STOP` |
| `01` | 任务一 | `run_task1_ab()` |
| `02` | 任务二 | `run_task2_abcd()` |
| `03` | 任务三 | `run_race_laps(1U)` |
| `04` | 任务四 | `run_race_laps(4U)` |
| `05` | PID 测试 | `run_motor_pid_stream()` |
| `06` | 转向测试 | `run_task6_ac_c_turn_test()` |
| `07` | PD 测试 | `run_motor_pd_stream()` |
| `10` | 航向标定 | `run_task10_ab_zero_test()` |

**命令格式**：

- 支持 HEX 和 ASCII（如 `03` 或 `0x03`）
- 运行中发送 `00` 或 `0x00` 会请求停车

## 2. 日志字段

### 2.1 通用字段

| 字段 | 含义 |
|------|------|
| `dist` / `phase_dist` | 当前段里程计数 |
| `yaw` / `rel_cdeg` | JY62 相对航向（0.01度） |
| `raw` / `mask` / `cnt` | 红外原始值、黑线掩码、命中数量 |
| `err` / `filt` / `der` | 红外误差、滤波误差、误差变化 |
| `line_turn` / `nav_turn` / `turn` | 红外修正、惯导修正、最终转向量 |
| `tdiff` / `ff` / `fb` / `corr` | 差速目标、前馈、反馈和合成修正 |
| `pwm=B/A` | B 轮和 A 轮 PWM |

### 2.2 停止原因

| 字段 | 含义 |
|------|------|
| `reason=point` | 正常点位触发 |
| `reason=force` | 距离保护触发 |
| `reason=uart_stop` | 串口停止触发 |
| `reason=timeout` | 超时停止 |

### 2.3 任务三/四特有字段

| 字段 | 含义 |
|------|------|
| `seg` | 当前段（AC/CB/BD/DA） |
| `lap` | 当前圈数 |
| `phase` | 阶段索引 |
| `h_raw` | 当前航向 |
| `h_tgt` | 目标航向 |
| `h_corr` | 航向修正 |
| `exp` | 预期弧线角 |
| `herr` | 航向误差 |
| `valid` | 红外有效标志 |
| `seen` | 本段见过线标志 |

## 3. 日志开关

```c
#define TASK2_RAM_LOG_ENABLE          (1)   // Task2 RAM 日志
#define RACE_UART_LOG_ENABLE          (0)   // 实时文本日志
#define RACE_RAM_LOG_ENABLE           (0)   // RAM dump 日志
#define RACE_REALTIME_EVENT_LOG_ENABLE (0)  // 实时事件日志
```

**说明**：

- 打开 RAM 日志后，输出前缀是 `RACE_RAM_BEGIN` 到 `RACE_RAM_END`
- 实时文本日志用于调试，RAM 日志用于详细分析

## 4. 跑车流程

### 4.1 准备阶段

1. 上电等待 `TASK ready` 输出
2. 确认串口连接正常
3. 准备好记录日志

### 4.2 航向标定

1. 车头朝 AB 方向放置
2. UART0 发送 `10`
3. 确认输出 `TASK10 zero_ab: ok=1 nav=1 rel=0`
4. 等待进入持续监控模式

### 4.3 启动任务

1. 把车放到任务起点
2. 发送任务命令（`01`-`04`）
3. 观察串口日志输出

### 4.4 运行监控

观察以下日志：

- `start`：任务启动
- `RACE_DATA`：周期数据
- `RACE_POINT_STATE`：点位触发
- `stop: reason=...`：停止原因

### 4.5 停车

- 自动停车：任务完成后自动刹车
- 手动停车：发送 `00` 或 `0x00`

## 6. 常见问题

### 6.1 串口无输出

- 检查 UART0 连接
- 确认波特率 115200
- 检查电源

### 6.2 任务不启动

- 确认发送的是 `01`-`04`
- 检查是否发送了 `00` 停车
- 确认 `ENABLE_CONTEST_TASKS=1`

### 6.3 日志不完整

- 使用 `--strict` 模式检查
- 确认日志文件编码为 UTF-8
- 检查是否有丢帧
