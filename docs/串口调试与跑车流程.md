# 串口调试与跑车流程

本文档记录当前这版程序怎么通过串口观察数据，以及什么时候让小车真正跑起来。等待蓝牙串口模块到货前，可以继续使用现有串口方式调试；蓝牙模块到货后，优先先验证串口链路，再改控制逻辑。

## 1. 当前串口配置

UART0 由 `main.syscfg` 配置，当前参数如下：

| 项目 | 当前值 |
|---|---|
| 串口实例 | `UART_0` |
| 发送引脚 | `PA28 / UART0_TX` |
| 接收引脚 | `PA31 / UART0_RX` |
| 波特率 | `115200` |
| 数据格式 | 8 数据位，无校验，1 停止位 |

接线时按交叉连接：

| MSPM0G3507 | 串口模块 |
|---|---|
| `PA28 / TX` | 模块 `RX` |
| `PA31 / RX` | 模块 `TX` |
| `GND` | 模块 `GND` |

注意：如果后续同时接 DAPLink 串口和蓝牙串口，不要让两个模块的 `TX` 同时直接驱动 MSPM0 的 `PA31/RX`。最稳妥的方式是同一时间只接一个发送源。

## 2. 串口打印函数

当前工程主要使用 `Board/board.c` 里的：

```c
lc_printf("hello\r\n");
```

它会通过 UART0 阻塞式发送字符串。阻塞式打印简单可靠，但打印太频繁会影响主循环速度，所以当前程序：

- 红外测试每 `100ms` 打印一次。
- 直行 PID 每 `100ms` 打印一次。

## 3. 模式一：八路红外循迹模块测试

当前默认就是这个模式：

```c
#define ENABLE_IR_TRACKING_UART_TEST (1)
```

这个模式用于验证：

- I2C 接线是否正确。
- 八路红外模块是否有数据。
- 黑线和白底逻辑是否正确。
- X1 到 X8 的左右方向是否正确。

程序现象：

- TB6612 保持刹车。
- 小车不会主动跑。
- 串口持续打印红外模块数据。

典型输出：

```text
IR raw=0xE7 mask=0x18 count=2 lost=0 pos=0 err=0
```

判断方法：

| 现象 | 说明 |
|---|---|
| 中间压线时 `err=0` | 方向和中心正常 |
| 黑线往左时 `err` 变负 | 左右方向正常 |
| 黑线往右时 `err` 变正 | 左右方向正常 |
| 一直 read failed | 优先查电源、共地、SDA/SCL、I2C 地址 |
| `mask=0x00` 且 `lost=1` | 8 路都没看到黑线 |

## 4. 模式二：直行 PID 跑车测试

如果要让小车跑直行 PID，把开关改为：

```c
#define ENABLE_IR_TRACKING_UART_TEST (0)
```

建议步骤：

1. 先把小车架空。
2. 编译并烧录。
3. 看左右轮是否都向前转。
4. 看串口 `B_spd` 和 `A_spd` 是否接近。
5. 确认无异常后再让小车落地低速跑。

当前基础 PWM 在 `main.c` 中：

```c
#define STRAIGHT_B_BASE_PWM (300)
#define STRAIGHT_A_BASE_PWM (310)
```

其中：

- B 电机是左轮。
- A 电机是右轮。
- A 轮基础 PWM 稍大，是为了补偿实车左右差异。

典型输出：

```text
PID t=1000 B_cnt=56 A_cnt=55 B_spd=56 A_spd=55 err=1 P=16 I=-65 D=0 corr=-2 B_pwm=302 A_pwm=308
```

字段含义：

| 字段 | 含义 |
|---|---|
| `t` | 程序运行时间，单位 ms |
| `B_cnt` | 最近一个控制周期内 B 电机编码器增量 |
| `A_cnt` | 最近一个控制周期内 A 电机编码器增量 |
| `B_spd` | B 电机简化速度估计，当前等于 `abs(B_cnt)` |
| `A_spd` | A 电机简化速度估计，当前等于 `abs(A_cnt)` |
| `err` | `B_spd - A_spd` |
| `P/I/D` | PID 三项的未缩放中间值 |
| `corr` | 缩放和限幅后的 PWM 修正量 |
| `B_pwm` | 最终给左轮 B 电机的 PWM |
| `A_pwm` | 最终给右轮 A 电机的 PWM |

`err` 判断：

| `err` | 含义 | 程序动作 |
|---|---|---|
| 大于 0 | 左轮 B 更快 | 降低 B PWM，提高 A PWM |
| 小于 0 | 右轮 A 更快 | 提高 B PWM，降低 A PWM |
| 接近 0 | 左右轮速度接近 | 修正量较小 |

## 5. 编码器自检

`main.c` 中还有编码器自检开关：

```c
#define ENABLE_ENCODER_SELF_TEST (0)
```

只有小车架空时才建议改成 1。自检会分别让 B 电机、A 电机短时间转动，并检查对应编码器有没有计数。

如果车放在地上打开自检，小车可能突然移动，所以默认保持 0。

## 6. 蓝牙串口模块到货后的测试顺序

建议不要一上来就改完整控制逻辑，先按下面顺序做：

1. 只接蓝牙串口模块的 `VCC/GND/TX/RX`。
2. 保持 `ENABLE_IR_TRACKING_UART_TEST` 为 1。
3. 手机或电脑连接蓝牙串口，确认能看到红外模块输出。
4. 再把 `ENABLE_IR_TRACKING_UART_TEST` 改为 0，确认能看到 PID 输出。
5. 最后再考虑增加串口命令，例如启动、停车、修改基础 PWM。

建议先实现几个非常简单的命令：

```text
S  启动
P  停车
I  打印一次红外数据
+  基础 PWM 加一点
-  基础 PWM 减一点
```

命令解析确认稳定后，再做更复杂的参数调试协议。

## 7. 烧录前检查清单

每次烧录前建议快速检查：

- `ENABLE_IR_TRACKING_UART_TEST` 是否是你想要的模式。
- 小车是否架空，或者电机电源是否暂时断开。
- TB6612 的 `VM` 电机电源和主控 `GND` 是否共地。
- 八路红外模块是否已校准，离地高度是否稳定。
- 串口工具是否设置为 `115200 8N1`。
- 新增 `.c` 文件后是否在 CCS 里执行过 `Refresh` 和 `Clean Project`。

## 8. 常见问题

### 串口没有输出

优先检查：

- 是否已经调用 `SYSCFG_DL_init()`。
- 串口工具波特率是否为 `115200`。
- `TX/RX` 是否接反。
- 是否有多个串口工具同时占用同一个端口。
- 蓝牙串口模块是否已经配对并打开正确的 COM 口。

### 红外读取失败

优先检查：

- `SDA=PB3`、`SCL=PB2` 是否接对。
- `VCC/GND` 是否接好。
- I2C 地址是否为 `0x12`。
- 模块 I2C 上拉电压是否超过 MSPM0 输入可接受范围。

### 小车只接电脑时能跑，脱离 DAPLink 不跑

优先检查：

- 主控板是否有独立供电。
- 主控 `GND`、TB6612 `GND`、电池负极是否共地。
- TB6612 `VCC` 是否接到主控逻辑电源。
- `STBY` 是否被程序拉高。
- 电机电源 `VM` 是否单独供电并有足够电流。

### 烧录时报 flash 验证错误

可以先尝试：

- 断电重插开发板。
- CCS 里做一次全片擦除或重新 Debug。
- 检查是否有程序正在占用调试器。
- 确认供电稳定，避免烧录过程中掉电。

如果仍然反复出现，再单独排查调试器、驱动和板卡连接。
