# 串口调试与跑车流程

本文记录当前版本如何通过 UART0 控制任务、观察串口日志，并根据字段判断小车状态。当前调试串口为 UART0，`115200 8N1`。

## 1. 串口配置

| 项目 | 当前值 |
|---|---|
| 串口实例 | `UART_0` |
| 发送引脚 | `PA28 / UART0_TX` |
| 接收引脚 | `PA31 / UART0_RX` |
| 波特率 | `115200` |
| 数据格式 | 8 数据位，无校验，1 停止位 |

接线时交叉连接：

| MSPM0G3507 | 串口模块 |
|---|---|
| `PA28 / TX` | 模块 `RX` |
| `PA31 / RX` | 模块 `TX` |
| `GND` | 模块 `GND` |

如果同时接 DAPLink 串口和蓝牙串口，不要让两个外部 TX 同时直接驱动 MSPM0 的 `PA31/RX`。

## 2. UART0 命令

当前解析器支持三类输入：

- 人手串口工具常用 ASCII：`01`、`02`、`10`、`11`。
- 带前缀 ASCII：`t01`、`t10`、`T11`。
- 原始 HEX 字节：`0x01..0x07`、`0x10`、`0x11`、`0x00`。

| 命令 | 作用 |
|---|---|
| `00` | 运行中强制停车 |
| `01` | 启动任务一 A -> B 直线 |
| `02` | 启动任务二 A -> B -> C -> D -> A |
| `03` | 启动任务三 A -> C -> B -> D -> A |
| `04` | 启动任务四，按任务三路线连续 4 圈 |
| `05` | 启动 PID 轮速/编码器/PWM 测试 |
| `06` | 启动 A->C 后 C 点快速左转测试 |
| `07` | 启动 PD 轮速/编码器/PWM 测试 |
| `10` | Task10，AB 方向零航向标定 |
| `11` | Task11，全程黑线经验采集 |

任务命令被接受后会打印：

```text
TASK UART command accepted: id=...
```

上电后会打印：

```text
TASK ready: buttons A26/A24/B24/A22 or UART0 HEX bytes 01..07,10,11; 00=stop while running; ASCII t01..t11 still ok
```

## 3. 实体按键

| 按键引脚 | 任务 |
|---|---|
| `A26 / PA26` | 任务一 |
| `A24 / PA24` | 任务二 |
| `B24 / PB24` | 任务三 |
| `A22 / PA22` | 任务四 |

四个按键均按低电平按下处理。任务开始前会消抖，再进入对应任务。

## 4. JY62 日志

典型输出：

```text
JY62 mode=TASK1_AB t=500 df=... ok=1 flags=0x07 yaw_cdeg=... rel_cdeg=... gz_mdps=... gzlp_mdps=... rx=... head=... frames=... err=0/0/0
```

| 字段 | 含义 |
|---|---|
| `mode` | 当前场景，如 `boot_zero`、`TASK1_AB`、`TASK10_AB_ZERO` |
| `t` | 当前场景下时间，单位 ms |
| `df` | 距离上次读取新增的有效帧数 |
| `ok` | 导航数据是否有效 |
| `flags` | 最近更新的数据类型标志，常见 `0x07` |
| `yaw_cdeg` | JY62 原始 yaw，单位 0.01 度 |
| `rel_cdeg` | 相对最近一次置零的 yaw，单位 0.01 度 |
| `gz_mdps` | Z 轴角速度，单位 mdps |
| `gzlp_mdps` | 低通后的 Z 轴角速度 |
| `rx/head/frames` | UART1 接收字节数、帧头数、有效帧数 |
| `err` | `校验错误/UART错误/溢出` 计数 |

判断：

- `ok=1` 且 `frames` 增长：JY62 正常。
- `err` 持续增长：优先查线序、波特率、供电和共地。
- 任务开始置零后 `rel_cdeg` 应接近 0。

## 5. 任务一/任务二直线日志

任务一和任务二 AB/CD 常见日志来自 `run_straight_to_line_segment()`：

```text
TASK1_AB start: ctrl=task5 zero=1 ycorr=1 ...
TASK1_AB t=... dist=... arm=... raw=0x.. mask=0x.. cnt=... lost=... ir=... nav=... yaw=... gzlp=... ycorr=... B_total=... A_total=... d_err=... d_corr=... B_spd=... A_spd=... v_tgt=... v_err=... P=... I=... D=... ff=... fb=... corr=... B_pwm=... A_pwm=...
TASK1_AB stop: reason=line ...
```

| 字段 | 含义 |
|---|---|
| `dist` | 当前段距离估计，`(abs(B_total)+abs(A_total))/2` |
| `arm` | 是否允许红外到线触发 |
| `raw/mask/cnt/lost` | 红外原始值、黑线掩码、触发探头数、是否丢线 |
| `yaw` | 当前 JY62 相对航向 |
| `ycorr` | JY62 航向修正量 |
| `B_total/A_total` | 左右轮当前段累计编码器计数 |
| `d_err/d_corr` | 累计距离误差和距离修正 |
| `B_spd/A_spd` | 左右轮当前周期速度估计 |
| `v_tgt/v_err` | 目标轮速差和 PID 误差 |
| `P/I/D` | 速度 PID 三项中间值 |
| `ff/fb/corr` | 前馈、反馈和最终修正 |
| `B_pwm/A_pwm` | 最终左轮 B、右轮 A PWM |

`corr > 0` 表示降低 B 轮、提高 A 轮；`corr < 0` 表示提高 B 轮、降低 A 轮。

## 6. 任务三/任务四直线日志

任务三/四 AC、BD 使用 `run_heading_straight_to_line_segment()`，日志中会出现：

```text
TASK3_AC start: zero=0 heading_only=1 fast=0 line_protect=2 ...
TASK3_BD target: B_exit=... target=-13920 mode=absolute
TASK3_BD t=... dist=... h_raw=... h_tgt=... h_flt=... h_use=... h_corr=... search=... corr=... base=... B_pwm=... A_pwm=...
```

| 字段 | 含义 |
|---|---|
| `heading_only` | 是否主要按目标航向控制 |
| `line_protect` | 入线/寻线保护模式 |
| `h_raw` | 当前 JY62 相对航向 |
| `h_tgt` | 当前直线目标航向 |
| `h_flt/h_use` | 航向滤波值和实际使用值 |
| `h_gain/h_wob` | 晃动降权后的增益和晃动标志 |
| `h_corr` | 航向修正量 |
| `search` | 寻线辅助修正量 |
| `corr` | 最终综合修正量 |
| `base` | 当前基础 PWM，格式 `B/A` |

当前任务三/四中，AC 和 BD 都使用固定绝对目标角：

```text
AC target = -3660
BD target = -13920
```

## 7. 弧线日志

任务三/四当前主要弧线日志来自 `run_task3_task11_arc_line_follow_segment()`：

```text
TASK3_CB start: ctrl=task11_arc dir=-1 final=0 base=540 diff=-44/-44 ...
TASK3_CB t=... dist=... yaw=... yprog=... exp=... herr=... gzlp=... ir=... valid=... seen=... raw=0x.. mask=0x.. cnt=... err=... filt=... line_turn=... nav_turn=... turn=... tdiff=... ff=... fb=... corr=... pwm=...
TASK3_CB stop: ctrl=task11_arc reason_id=3 reason=arc_done ...
```

| 字段 | 含义 |
|---|---|
| `dir` | 弧线方向，`-1` 左弧，`1` 右弧 |
| `base` | 弧线基础 PWM |
| `diff` / `tdiff` | 入弯/巡航目标轮速差 |
| `yaw/yprog` | 当前航向和本段转过角度 |
| `exp` | 按里程计算出的理论弧线航向 |
| `herr` | 实际弧线航向相对理论值的误差 |
| `valid/seen` | 当前红外是否有效、当前段是否见过线 |
| `err/filt` | 红外误差和滤波误差 |
| `line_turn` | 红外 PD 给出的转向 |
| `nav_turn` | JY62 弧线模型给出的转向 |
| `turn` | 最终转向量 |
| `ff/fb/corr` | 差速控制前馈、反馈和综合修正 |
| `pwm` | 最终左右轮 PWM，格式 `B/A` |

## 8. Task10 和 Task11

### 8.1 Task10

发送 `10` 后，车头朝 AB 方向时执行零航向标定。正常日志：

```text
TASK UART command accepted: id=10
JY62 mode=TASK10_AB_ZERO ...
TASK10 zero_ab: ok=1 nav=1 rel=0 gzlp=...
```

`ok=1 nav=1 rel=0` 表示标定成功。

当前任务三/四使用固定绝对目标角，建议在跑 `03/04` 前先执行 Task10。否则 `-3660`、`-13920` 这些目标角会相对上一次 JY62 零点，实车方向会整体偏掉。

### 8.2 Task11

Task11 需要车头从 A 指向 C，AC 和 BD 都贴黑线。阶段顺序：

```text
AC -> CB -> BD -> DA
```

如果打开 `TASK11_UART_LOG_ENABLE`，运动期间会输出 `TASK11_DATA` 和 `TASK11_POINT`。如果打开 `TASK11_RAM_LOG_ENABLE`，运动结束后会输出：

```text
TASK11_RAM_BEGIN
TASK11_CFG
TASK11_DUMP_SECTION name=EVT
TASK11_DUMP_SECTION name=SUM
TASK11_DUMP_SECTION name=WIN
TASK11_RAM_END
```

当前默认这些大日志开关是关闭的，目的是减少串口输出和 SRAM 占用。

## 9. 05/07 轮速测试

| 命令 | 模式 |
|---|---|
| `05` | PID 轮速测试 |
| `07` | PD 轮速测试 |

用途：

- 检查 A/B 电机是否都转。
- 检查左右编码器是否都计数。
- 标定 `COUNTS_PER_CM`。
- 判断某次突然转圈是否为电机线、编码器线或 TB6612 接线松动。

发送 `00` 可以退出测试。

## 10. 常见问题

| 现象 | 优先检查 |
|---|---|
| 串口没有输出 | 波特率、TX/RX 交叉、串口占用、供电 |
| `ir=0` | 红外模块供电、共地、SDA/SCL、I2C 地址 |
| `nav=0` 或 JY62 `frames` 不增长 | JY62 供电、共地、TX/RX、波特率 |
| 一侧 `B_spd/A_spd` 长期为 0 | 电机线、编码器线、TB6612 方向脚 |
| `reason=force` | 到点线没扫到，检查角度、红外高度、arm 距离和寻线 |
| `corr` 或 `turn` 长期贴近限幅 | 先查硬件，再调 PWM/PID/航向权重 |
| 起步突然大转向 | 检查电机线松动、JY62 是否被手碰到、起步置零是否稳定 |
