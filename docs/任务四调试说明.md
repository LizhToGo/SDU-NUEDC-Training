# 任务四调试说明

入口：A22 按键或 UART0 `04`

任务四用于在任务三路线基础上连续跑 4 圈，并使用单独的高速参数做竞速调试。当前分支是有待完善的测试分支，实车调参时优先保留 RAM 日志。

## 1. 启动姿态

当前任务四以 Task10 的 AB 零航向为基准：

1. 车头朝 AB 方向放置。
2. UART0 发送 `10`，确认 JY62 相对角已经归零。
3. 把车放到 A 点，车头仍朝 B 方向。
4. UART0 发送 `04` 或按 A22 启动任务四。

启动后会先按 `RACE_TASK4_START_ALIGN_ENABLE` 执行右转对准 AC 目标角，再进入 AC 直线。

## 2. 当前路线

```text
AC 直线 -> CB 左弧线 -> BD 直线 -> DA 右弧线
```

任务四执行 4 圈；前三圈 DA 到 A 后继续下一圈，第四圈 DA 到 A 后停车。

## 3. 控制逻辑

- AC/BD 直线：惯导航向控制为主，红外只用于点位识别和日志，不参与直线纠偏。
- CB/DA 弧线：红外循线和差速控制为主，JY62 用于记录和辅助判断。
- C/D 点入弯：直线段扫到红外线且过了对应 arm 距离后入弯。
- AC/BD 漏线：到强制距离后先转向弧线方向，再直行找线。
- B/A 出弯：弧线出口后按绝对目标角执行 `race_gyro_turn_to_yaw()`。

## 4. 关键参数

```c
#define TASK4_LAP_COUNT                  (4)
#define RACE_TASK4_CONTROL_PERIOD_MS     (12)
#define RACE_TASK4_STRAIGHT_BASE_PWM     (840)
#define RACE_TASK4_LINE_MAX_PWM          (900)
#define RACE_TASK4_ARC_BASE_PWM          (670)
#define RACE_TASK4_AC_HEADING_TARGET_CDEG (-3300)
#define RACE_TASK4_BD_HEADING_TARGET_CDEG (-18000 + 3800)
#define RACE_TASK4_BD_POINT_ARM_COUNT    (6600)
#define RACE_TASK4_BD_FORCE_TURN_COUNT   (7200)
#define RACE_TASK4_AC_FORCE_TURN_COUNT   (7520)
#define RACE_TASK4_FIRST_AC_FORCE_TURN_COUNT (7800)
#define RACE_TASK4_EXIT_TURN_PREDICT_MS  (40)
```

任务三仍使用自己的稳定参数。除非明确说明，调任务四时不要顺手改任务三参数。

## 5. RAM 日志判断

任务四建议优先看 `RACE_RAM_BEGIN` 到 `RACE_RAM_END` 的 RAM dump：

- `RACE_SEG`：每段汇总，`ph=0/1/2/3` 分别是 AC/CB/BD/DA。
- `rsn=1`：正常点位触发。
- `rsn=2`：距离强制触发。
- `RACE_WIN`：点位窗口采样，重点看 `dist/yaw/exp/herr/mask/cnt/fl`。
- `RACE_WIN_NAV`：对应窗口的导航和转向量，重点看 `nav/turn/gz/gzl`。

判断“红外扫到了但没有入弯”时，看 `RACE_WIN` 中第一次 `mask != 0x00` 或 `cnt > 0` 的距离是否已经超过该段 arm：

- AC 正常点位 arm：`RACE_AC_POINT_ARM_COUNT`
- BD 任务三 arm：`RACE_BD_POINT_ARM_COUNT`
- BD 任务四 arm：`RACE_TASK4_BD_POINT_ARM_COUNT`

如果扫线距离小于 arm，代码会认为还没到点位，继续直行；这时应调小对应 arm，而不是先改转向角。

## 6. 常见问题

### 6.1 AC 冲过 C 线

先看 AC 段 `RACE_WIN`：

- `mask/cnt` 出现在线上，但 `dist < RACE_AC_POINT_ARM_COUNT`：AC arm 太晚。
- `mask/cnt` 一直没有出现，且航向接近 `exp`：优先调 AC 目标角或摆放姿态。
- `rsn=2`：走到了强制距离，说明正常点位没有成功触发。

### 6.2 BD 冲过 D 线

重点看 BD 段：

- `mask/cnt` 出现且 `dist < RACE_TASK4_BD_POINT_ARM_COUNT`：任务四 BD arm 太晚。
- `dist` 接近 `RACE_TASK4_BD_FORCE_TURN_COUNT` 才触发：兜底距离偏晚。
- 刚进入 BD 时 `yaw` 已接近 `exp`，但随后大幅摆动：更像出弯动态问题，不是目标角没到。

### 6.3 出弯角度不准

B 点出弯目标由 `RACE_TASK4_BD_HEADING_TARGET_CDEG` 控制，A 点出弯目标由 `RACE_TASK4_AC_HEADING_TARGET_CDEG` 控制。任务四启用了 `RACE_TASK4_EXIT_TURN_PREDICT_ENABLE`，预测时间是 `RACE_TASK4_EXIT_TURN_PREDICT_MS`。

### 6.4 日志太少

确认：

- `RACE_RAM_LOG_ENABLE`
- `RACE_DUMP_SUM_ENABLE`
- `RACE_DUMP_EVT_ENABLE`
- `RACE_DUMP_WIN_ENABLE`
- `RACE_DUMP_WIN_STRIDE`

`WIN` 只保存点位附近窗口，不是全程实时日志；需要全程实时观察时再临时打开实时日志，测试完关掉。

## 7. 推荐调参顺序

1. 先确认 AC/BD 直线航向是否接近目标角。
2. 再看红外扫线距离是否早于 arm。
3. 若扫到线但没有入弯，先调对应 arm。
4. 若完全扫不到线，再调目标角或强制转向角。
5. 若出弯后摆动，优先看出弯速度、预测停止和出弯后 800 计数内的航向修正。
