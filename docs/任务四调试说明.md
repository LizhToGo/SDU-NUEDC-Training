# 任务四调试说明

本文记录当前任务四的实现逻辑、串口字段和后续调参重点。当前版本用于 push 留档：代码已经能按任务三路线连续跑 4 圈，属于基础可跑版本，后续还需要继续优化稳定性和用时。

## 1. 任务目标

题目要求：

```text
按要求（3）的路径自动行驶 4 圈停车，用时越少越好。
```

当前任务四路线为：

```text
A -> C -> B -> D -> A
```

连续执行 4 圈。前三圈经过 A 点时不停车，第四圈回到 A 点后停车并触发结束声光。

## 2. 启动方式

| 入口 | 作用 |
|---|---|
| `A22 / PA22` | 启动任务四 |
| UART0 `04` | 启动任务四 |
| UART0 `00` | 任务运行中强制停车 |

启动前摆车方式和任务三一致：把小车放在 A 点，车头指向 C 点。第一圈 AC 开始时会把当前 JY62 航向置零。

## 3. 当前关键参数

```c
#define TASK4_LAP_COUNT               (4)
#define TASK4_AC_RELATIVE_TURN_CDEG   (-TASK3_BD_RELATIVE_TURN_CDEG)
#define TASK4_AC_LINE_SEARCH_PROTECT  (3)
```

其中任务三已经调好的参数为：

```c
#define TASK3_BD_RELATIVE_TURN_CDEG   (6700)
```

因此当前：

```text
B 点出线后：B_exit + 6700
A 点换圈后：A_exit - 6700
```

这里的单位都是 0.01 度，`6700` 表示 `67.00` 度。

## 4. 分段逻辑

每一圈由四段组成：

```text
TASK4_Lx_AC: A -> C 直线
TASK4_Lx_CB: C -> B 任务三专用红外弧线
TASK4_Lx_BD: B -> D 相对航向直线
TASK4_Lx_DA: D -> A 任务三专用红外弧线
```

其中 `x` 是圈数，范围为 `1..4`。

### 4.1 第一圈 AC

第一圈 AC 与任务三 AC 基本一致：

- 触发启动声光。
- JY62 当前方向置零。
- 使用直线段控制。
- 接近 C 点后开启红外寻线。
- 红外识别到 C 点入线后触发 C 点声光，进入 CB。

### 4.2 CB 弧线

CB 使用任务三专用弧线逻辑：

- 入弯初期降低速度并加强转向。
- 中段主要依靠红外误差循迹。
- B 点出线需要满足弧线进度、B 点方向条件和红外出线事件。
- B 点出线后不刹车，只触发声光，然后进入 BD。

### 4.3 BD 直线

BD 沿用任务三已经调通的相对转角：

```text
BD target = B_exit + TASK3_BD_RELATIVE_TURN_CDEG
```

当前 `TASK3_BD_RELATIVE_TURN_CDEG = 6700`。

BD 直线段使用 `heading_only=1`，优先按照目标航向边走边转。到 D 点附近后红外扫到线，触发 D 点声光并进入 DA。

### 4.4 DA 弧线和 A 点换圈

DA 也使用任务三专用弧线逻辑。

前三圈：

- A 点出线后不刹车。
- 触发一次 A 点声光。
- 读取 A 点出线瞬间航向 `A_exit`。
- 计算下一圈 AC 目标：

```text
next AC target = A_exit + TASK4_AC_RELATIVE_TURN_CDEG
```

当前 `TASK4_AC_RELATIVE_TURN_CDEG = -6700`，也就是与 B 点转向等大反向。

第四圈：

- A 点出线后刹车。
- 触发结束声光。
- 重置编码器距离计数。

## 5. 关键串口日志

任务四启动：

```text
TASK4 start: task3 route x4, B_rel=6700, A_rel=-6700
```

每圈开始：

```text
TASK4 lap=1 start: AC_target=0 zero=1 final=0
TASK4 lap=2 start: AC_target=... zero=0 final=0
```

B 点目标角：

```text
TASK4_L1_BD target: lap=1 B_exit=... rel_turn=6700 target=... mode=relative
```

A 点换圈目标角：

```text
TASK4_A target seed: lap=1 A_exit=... rel_turn=-6700 next_target=... mode=relative
```

最终完成：

```text
TASK4 complete: 4 laps stopped at A, distance reset
```

## 6. 调参判断

### 6.1 第二圈 AC 到不了 C

优先看：

```text
TASK4_A target seed: lap=1 A_exit=... rel_turn=-6700 next_target=...
TASK4_L2_AC t=... h_raw=... h_tgt=... h_corr=...
TASK4_L2_AC stop: reason=...
```

判断：

- `h_raw` 能靠近 `h_tgt`，但仍扫不到 C：优先调 A 点相对转角或 AC 寻线距离。
- `h_raw` 长期偏离 `h_tgt`：看 JY62 是否有效、`h_corr` 是否贴限幅、电机线是否松动。
- `reason=force`：说明没扫到 C 点线，先别改弧线参数。

### 6.2 B 点到 D 不稳定

任务三已确认 `6700` 基本可用。任务四中如果某圈 BD 失误，先看该圈 B 点出线姿态：

```text
TASK4_Lx_BD target: lap=x B_exit=... rel_turn=6700 target=...
```

如果 `B_exit` 本身偏得很大，问题可能来自前一段 CB 出线姿态，而不是 `6700` 本身。

### 6.3 A 点停车滞后

第四圈停车由 DA 弧线的最终 A 点出线事件触发。如果声光晚响或车多跑一段，优先看：

```text
TASK4_L4_DA stop: reason_id=... reason=...
```

期望最终为红外出线触发，而不是距离兜底。

## 7. 当前留档结论

当前版本的任务四已经具备完整四圈流程：

- 第一圈按任务三方式启动并置零。
- B 点使用 `+6700` 相对转角。
- A 点换圈使用 `-6700` 相对转角。
- 前三圈不停，第四圈 A 点停车。

后续优化建议先围绕 A 点换圈后的第二圈 AC 稳定性调参，再优化速度。
