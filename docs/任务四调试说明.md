# 任务四调试说明

本文记录当前任务四的实现逻辑、串口字段和后续调参重点。任务四入口为 `A22 / PA22` 或 UART0 `04`。

**commit 0199d70 变更**：任务四入口改走 `run_task11_ir_map_test_laps(4U)`，B/A 点出口转向改为 `gyro_turn_to_yaw`，Task11 使用独立航向目标。

## 1. 任务目标

题目要求：

```text
按任务三路径自动行驶 4 圈停车，用时越少越好。
```

当前任务四路线为：

```text
A -> C -> B -> D -> A
```

连续执行 4 圈。前三圈经过 A 点时不结束，第四圈回到 A 点后停车并触发结束声光。

## 2. 当前关键变化

当前代码已经不是旧的相对转角方案。

旧方案：

```text
B 点：B_exit + TASK3_BD_RELATIVE_TURN_CDEG
A 点：A_exit + TASK4_AC_RELATIVE_TURN_CDEG
```

当前方案（commit 0199d70 后）：

```text
AC target = TASK11_AC_HEADING_TARGET_CDEG  (-50)
BD target = TASK11_BD_HEADING_TARGET_CDEG  (-10638)
B/A 点出口转向 = gyro_turn_to_yaw（陀螞仪绝对航向转向）
```

这些固定目标角默认以 Task10 建立的 AB 零航向为基准。跑任务四前建议先让车头朝 AB，发送 `10`，确认 `rel=0` 后再把车放到 A 点、车头朝 C 启动 `04`。

对应宏：

```c
#define TASK4_LAP_COUNT              (4)
#define TASK4_AC_HEADING_TARGET_CDEG TASK3_AC_HEADING_TARGET_CDEG  /* 保留，但 Task11 使用独立值 */
#define TASK4_BD_HEADING_TARGET_CDEG TASK3_BD_HEADING_TARGET_CDEG  /* 保留，但 Task11 使用独立值 */

/* Task11 专用航向目标（实际任务三/四使用这组） */
#define TASK11_AC_HEADING_TARGET_CDEG (-50)
#define TASK11_BD_HEADING_TARGET_CDEG (-10638)
```

实际值来自任务三：

```c
#define TASK3_AC_HEADING_TARGET_CDEG (-3660)
#define TASK3_BD_HEADING_TARGET_CDEG (-13920)
```

日志中会显示：

```text
TASK4 start: task3 route x4, AC_abs=-3660, BD_abs=-13920
TASK4_Lx_BD target: lap=x B_exit=... target=-13920 mode=absolute
TASK4_A target seed: lap=x A_exit=... next_target=-3660 mode=absolute
```

`A_exit/B_exit` 仍用于分析出弯姿态，但不再参与目标角计算。

## 3. 启动方式

| 入口 | 作用 |
|---|---|
| `A22 / PA22` | 启动任务四 |
| UART0 `04` | 启动任务四 |
| UART0 `00` | 任务运行中强制停车 |

启动前摆车方式和任务三一致：先用 Task10 标定 AB 零航向，再把小车放在 A 点，车头指向 C 点。当前任务四不会在第一圈 AC 内重新置零，依赖启动前已有的 JY62 相对航向状态。

## 4. 分段逻辑

每一圈由四段组成：

```text
TASK4_Lx_AC: A -> C 固定航向直线
TASK4_Lx_CB: C -> B Task11 风格左弧线
TASK4_Lx_BD: B -> D 固定航向直线
TASK4_Lx_DA: D -> A Task11 风格右弧线
```

其中 `x` 是圈数，范围为 `1..4`。

### 4.1 AC 直线

每圈 AC：

- 目标角为 `TASK4_AC_HEADING_TARGET_CDEG`。
- 使用 `run_heading_straight_to_line_segment()`。
- 接近 C 点后开启红外寻线保护。
- 红外扫到 C 点线后进入 CB。

第一圈会触发启动声光；后续圈不单独做启动声光。

### 4.2 CB 弧线

CB 使用 `run_task3_task11_arc_line_follow_segment()`：

- `dir=-1`，左弧线。
- 基础速度来自 `TASK11_ARC_BASE_PWM`。
- 目标轮速差来自 `TASK11_CB_ARC_*_TARGET_DIFF`。
- 红外 PD 和 JY62 弧线模型共同控制。
- B 点出线成功时 `reason=arc_done`。

B 点出线后触发声光，然后进入 BD。

### 4.3 BD 直线

BD：

- 目标角固定为 `TASK4_BD_HEADING_TARGET_CDEG`。
- 当前不是根据 B 点出线角动态计算。
- 到 D 点附近后由红外扫线进入 DA。

关键日志：

```text
TASK4_Lx_BD target: lap=x B_exit=... target=-13920 mode=absolute
TASK4_Lx_BD t=... h_raw=... h_tgt=... h_corr=...
TASK4_Lx_BD stop: reason=...
```

### 4.4 DA 弧线和 A 点

DA：

- `dir=1`，右弧线。
- 使用 Task11 风格弧线。
- `stop_on_final_line=1`。

前三圈：

- A 点出线后不结束。
- 触发一次声光。
- 打印 A 点出线航向和下一圈固定 AC 目标。
- 进入下一圈 AC。

第四圈：

- A 点出线后停车。
- 触发结束声光。
- 重置编码器距离计数。

## 5. 关键串口日志

任务四启动：

```text
TASK4 start: task3 route x4, AC_abs=-3660, BD_abs=-13920
```

每圈开始：

```text
TASK4 lap=1 start: AC_target=-3660 zero=0 final=0
TASK4 lap=4 start: AC_target=-3660 zero=0 final=1
```

B 点目标角：

```text
TASK4_L1_BD target: lap=1 B_exit=... target=-13920 mode=absolute
```

A 点换圈：

```text
TASK4_A target seed: lap=1 A_exit=... next_target=-3660 mode=absolute
```

最终完成：

```text
TASK4 complete: 4 laps stopped at A, distance reset
```

## 6. 调参判断

### 6.1 第二圈以后 AC 到不了 C

优先看：

```text
TASK4_A target seed: lap=... A_exit=... next_target=-3660 mode=absolute
TASK4_Lx_AC t=... h_raw=... h_tgt=... h_corr=...
TASK4_Lx_AC stop: reason=...
```

判断：

- `h_raw` 能靠近 `h_tgt`，但仍扫不到 C：优先调 AC 目标角或寻线距离。
- `h_raw` 长期偏离 `h_tgt`：看 JY62 是否有效、`h_corr` 是否贴限幅、电机线是否松动。
- `reason=force`：说明没扫到 C 点线，先别改弧线参数。

### 6.2 B 点到 D 不稳定

当前不再看 `rel_turn`。看该圈：

```text
TASK4_Lx_BD target: lap=x B_exit=... target=-13920 mode=absolute
```

如果 B 出线姿态 `B_exit` 很差，问题可能来自 CB；如果 B 出线姿态稳定但 BD 总偏同侧，调 `TASK3_BD_HEADING_TARGET_CDEG` 或其任务四别名。

### 6.3 CB/DA 弧线丢线

看：

```text
TASK4_Lx_CB t=... valid=... seen=... err=... line_turn=... nav_turn=... tdiff=... pwm=...
TASK4_Lx_DA t=... valid=... seen=... err=... line_turn=... nav_turn=... tdiff=... pwm=...
```

判断：

- `valid=0 seen=0`：入弯就没找到线。
- `valid=0 seen=1`：曾经在线上，后来丢线。
- `line_turn` 很大：红外误差大。
- `nav_turn` 很大：弧线实际角度相对模型偏差大。
- `tdiff` 方向错误：检查左右轮或弧线方向宏。

### 6.4 A 点停车滞后

第四圈停车由 DA 弧线最终 A 点事件触发。看：

```text
TASK4_L4_DA stop: ctrl=task11_arc reason_id=... reason=...
```

期望最终为 `reason=line`。如果是 `force` 或 `timeout`，说明最终 A 点出线没有被正常识别。

## 7. 当前调参建议

1. 先用任务三确认单圈路线完整。
2. 再跑任务四看第一圈到第二圈是否稳定切换。
3. 如果每圈误差同向积累，优先调整固定 AC/BD 目标角。
4. 如果只有弧线段中途丢线，优先调 Task11 风格弧线参数。
5. 如果某一点声光不触发，优先看对应段 `stop reason`，确认是红外点位没识别还是距离/角度门槛没打开。
