# 任务四说明

任务四在任务三路线基础上连续跑 4 圈，使用更激进的任务四专用参数：

```text
AC -> CB -> BD -> DA，共 4 圈
```

入口：

- 按键：`A22`
- UART0：`04`
- 函数：`run_race_laps(4U)`

## 起步与航向

任务四启动时会触发一次 ST011 声光提示。当前验收版不再保留 Task10，任务四使用起步时固件建立的 JY62 相对零点作为航向基准。

任务四对起步角度更敏感，摆车时要尽量按当前调参约定放置。

## 任务四专用参数

| 参数 | 含义 |
|---|---|
| `RACE_TASK4_CONTROL_PERIOD_MS` | 任务四控制周期 |
| `RACE_TASK4_STRAIGHT_BASE_PWM` | 任务四直线基础 PWM |
| `RACE_TASK4_LINE_MAX_PWM` | 任务四直线 PWM 上限 |
| `RACE_TASK4_ARC_BASE_PWM` | 任务四弧线基础 PWM |
| `RACE_TASK4_AC_HEADING_TARGET_CDEG` | 任务四 AC 目标航向 |
| `RACE_TASK4_BD_HEADING_TARGET_CDEG` | 任务四 BD 目标航向 |
| `RACE_TASK4_AC_FORCE_TURN_COUNT` | 任务四 AC 强制入弯距离 |
| `RACE_TASK4_FIRST_AC_FORCE_TURN_COUNT` | 任务四第一圈 AC 强制入弯距离 |
| `RACE_TASK4_BD_POINT_ARM_COUNT` | 任务四 BD 点位检测使能距离 |
| `RACE_TASK4_BD_FORCE_TURN_COUNT` | 任务四 BD 强制入弯距离 |
| `RACE_TASK4_EXIT_TURN_PREDICT_ENABLE` | 出弯预测停转 |

默认原则：调任务四时优先改 `RACE_TASK4_*`，不要顺手改任务三稳定参数。

## 入弯与出弯

- AC/BD 直线段主要依靠 JY62 目标航向和编码器差速闭环。
- 红外主要用于点位检测、强制入弯后的找线和 RAM 日志复盘。
- C/D 点入弯由 `race_sensor_fast_turn()` 控制。
- B/A 点出弯由 `race_gyro_turn_to_yaw()` 控制，任务四可启用预测停转。

## RAM 日志复盘

建议任务四高速调参时开启 `RACE_RAM_LOG_ENABLE`，关闭过密实时日志。

重点看：

- `RACE_CFG`：确认本次烧录参数。
- `RACE_SEG idx=N ph=0/1/2/3`：AC/CB/BD/DA 分段摘要。
- `RACE_SEG_LINE seen/first/last/lost`：红外是否扫到线、是否过早/过晚。
- `RACE_SEG_YAW yaw0/yaw1/ypr/herr`：航向是否到位、出弯是否过冲。
- `rsn=1`：正常点位触发。
- `rsn=2`：距离强制触发。

如果红外扫到线但没有入弯，先检查该段 `dist` 是否小于对应 `point_arm_count`；小于 arm 时线会被保护逻辑忽略。

## 验收前检查

1. `04` 能启动任务四。
2. ready 文案只显示 `01..04`。
3. RAM 日志容量足够覆盖 4 圈摘要。
4. 最近一版实车跑稳后再做速度提升。
