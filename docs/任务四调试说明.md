# 任务四调试说明

入口：A22 按键或 UART0 `04`

---

## 1. 任务目标

题目要求：

```text
按任务三路径自动行驶 4 圈停车，用时越少越好。
```

当前任务四路线：

```text
A -> C -> B -> D -> A（连续 4 圈）
```

---

## 2. 启动前准备

与任务三相同：先用 Task10 标定 AB 零航向。

1. 车头朝 AB 方向放置
2. UART0 发送 `10`，确认 `rel=0`
3. 把车放到 A 点，车头朝 C
4. 发送 `04` 启动

---

## 3. 关键变化

当前代码已经不是旧的相对转角方案。

**旧方案**：

```text
B 点：B_exit + TASK3_BD_RELATIVE_TURN_CDEG
A 点：A_exit + TASK4_AC_RELATIVE_TURN_CDEG
```

**当前方案**：

```text
AC target = RACE_AC_HEADING_TARGET_CDEG  (-50)
BD target = RACE_BD_HEADING_TARGET_CDEG  (-10638)
B/A 点出口转向 = race_gyro_turn_to_yaw（陀螺仪绝对航向转向）
```

这些固定目标角默认以 Task10 建立的 AB 零航向为基准。

---

## 4. 分段逻辑

每圈由四段组成：

```text
RACE_Lx_AC: A -> C 固定航向直线
RACE_Lx_CB: C -> B 竞速左弧线
RACE_Lx_BD: B -> D 固定航向直线
RACE_Lx_DA: D -> A 竞速右弧线
```

其中 `x` 是圈数，范围为 `1..4`。

### 4.1 AC 直线

每圈 AC：

- 目标角为 `TASK4_AC_HEADING_TARGET_CDEG`
- 使用 `run_heading_straight_to_line_segment()`
- 接近 C 点后开启红外寻线保护
- 红外扫到 C 点线后进入 CB

第一圈会触发启动声光；后续圈不单独做启动声光。

### 4.2 CB 弧线

CB 使用竞速弧线控制：

- `dir=-1`，左弧线
- 基础速度来自 `RACE_ARC_BASE_PWM`
- 目标轮速差来自 `RACE_CB_ARC_*_TARGET_DIFF`
- 红外 PD 和 JY62 弧线模型共同控制
- B 点出线成功时 `reason=arc_done`

B 点出线后触发声光，然后进入 BD。

### 4.3 BD 直线

BD：

- 目标角固定为 `TASK4_BD_HEADING_TARGET_CDEG`
- 当前不是根据 B 点出线角动态计算
- 到 D 点附近后由红外扫线进入 DA

**关键日志**：

```text
TASK4_Lx_BD target: lap=x B_exit=... target=-13920 mode=absolute
TASK4_Lx_BD t=... h_raw=... h_tgt=... h_corr=...
TASK4_Lx_BD stop: reason=...
```

### 4.4 DA 弧线和 A 点

DA：

- `dir=1`，右弧线
- 使用 Task11 风格弧线
- `stop_on_final_line=1`

**前三圈**：

- A 点出线后不结束
- 触发一次声光
- 打印 A 点出线航向和下一圈固定 AC 目标
- 进入下一圈 AC

**第四圈**：

- A 点出线后停车
- 触发结束声光
- 重置编码器距离计数

## 5. 关键串口日志

任务四启动：

```text
RACE start: ac_zero_collect laps=4 ...
```

每圈开始：

```text
RACE_DATA lap=0 seg=AC ...
RACE_DATA lap=3 seg=DA ...
```

B 点目标角：

```text
RACE_POINT_STATE seg=CB ...
```

A 点换圈：

```text
RACE_POINT_STATE seg=DA ...
```

最终完成：

```text
RACE complete: ac_zero_collect reason=point ...
```

## 7. 调参判断

### 7.1 第二圈以后 AC 到不了 C

优先看：

```text
RACE_DATA lap=... seg=AC ...
RACE_POINT_STATE seg=AC ...
```

判断：

- `h_raw` 能靠近 `h_tgt`，但仍扫不到 C：优先调 AC 目标角或寻线距离
- `h_raw` 长期偏离 `h_tgt`：看 JY62 是否有效、`h_corr` 是否贴限幅、电机线是否松动
- `reason=force`：说明没扫到 C 点线，先别改弧线参数

### 7.2 B 点到 D 不稳定

当前不再看 `rel_turn`。看该圈：

```text
RACE_DATA lap=... seg=BD ...
```

如果 B 出线姿态 `B_exit` 很差，问题可能来自 CB；如果 B 出线姿态稳定但 BD 总偏同侧，调 `TASK3_BD_HEADING_TARGET_CDEG` 或其任务四别名。

### 7.3 CB/DA 弧线丢线

看：

```text
RACE_DATA lap=... seg=CB ... err=... line_turn=... nav_turn=... tdiff=... pwm=...
RACE_DATA lap=... seg=DA ... err=... line_turn=... nav_turn=... tdiff=... pwm=...
```

判断：

- `valid=0 seen=0`：入弯就没找到线
- `valid=0 seen=1`：曾经在线上，后来丢线
- `line_turn` 很大：红外误差大
- `nav_turn` 很大：弧线实际角度相对模型偏差大
- `tdiff` 方向错误：检查左右轮或弧线方向宏

### 7.4 A 点停车滞后

第四圈停车由 DA 弧线最终 A 点事件触发。看：

```text
RACE_POINT_STATE seg=DA ...
```

期望最终为 `reason=line`。如果是 `force` 或 `timeout`，说明最终 A 点出线没有被正常识别。

---

## 8. 调参建议

1. 先用任务三确认单圈路线完整
2. 再跑任务四看第一圈到第二圈是否稳定切换
3. 如果每圈误差同向积累，优先调整固定 AC/BD 目标角
4. 如果只有弧线段中途丢线，优先调 Task11 风格弧线参数
5. 如果某一点声光不触发，优先看对应段 `stop reason`，确认是红外点位没识别还是距离/角度门槛没打开
