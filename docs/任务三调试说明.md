# 任务三调试说明

入口：B24 按键或 UART0 `03`

---

## 1. 路线目标

```text
A 点出发，车头朝 C
RACE_AC: A -> C 直线
RACE_CB: C -> B 左弧线
RACE_BD: B -> D 直线
RACE_DA: D -> A 右弧线
A 点停车
```

---

## 2. 启动前准备

由于使用固定绝对航向，跑任务三前建议先执行 Task10：

1. 车头朝 AB 方向放置
2. UART0 发送 `10`
3. 确认 `TASK10 zero_ab: ok=1 nav=1 rel=0`
4. 再把车放到 A 点，车头朝 C
5. 发送 `03` 启动

---

## 3. 分段逻辑

### 3.1 AC 直线

- **目标航向**：`TASK3_AC_HEADING_TARGET_CDEG`（-3660）
- **控制方式**：航向直线
- **结束条件**：扫到 C 点线

**重点字段**：

| 字段 | 含义 |
|------|------|
| `h_raw` | 当前 JY62 相对航向 |
| `h_tgt` | AC 固定目标角 |
| `h_corr` | 航向修正 |
| `search` | 接近 C 点后的寻线辅助 |
| `raw/mask/cnt` | C 点红外入线情况 |
| `reason=line` | 成功扫到 C 点线 |
| `reason=force` | 未扫到 C 点线，距离兜底 |

### 3.2 C 点

AC 扫到 C 点入线后：

- 触发一次 ST011 声光
- 不使用任务二旧弧线
- 进入 `RACE_CB` 左弧线

### 3.3 CB 弧线

CB 是左弧线，当前使用竞速弧线控制：

1. `straight_drive_update()` 根据目标轮速差输出基础左右 PWM
2. 红外有效时根据 `err/filt/der` 计算 `line_turn`
3. JY62 根据里程对应的理论弧线角计算 `nav_turn`
4. 最终 `turn = line_turn + nav_turn`
5. 当距离、弧线角度和点位红外条件满足时，以 `reason=arc_done` 出 B 点

**周期日志重点**：

| 字段 | 含义 |
|------|------|
| `dist` | 当前弧线段距离 |
| `yaw/yprog` | 当前航向 / 本段转过角度 |
| `exp` | 根据里程估算的理论弧线角 |
| `herr` | 实际角度相对理论值的误差 |
| `valid/seen` | 当前红外是否有效 / 本段是否见过线 |
| `err/filt` | 红外误差和滤波误差 |
| `line_turn` | 红外 PD 转向 |
| `nav_turn` | JY62 弧线模型转向 |
| `tdiff` | 目标轮速差 |
| `ff/fb/corr` | 差速前馈、反馈和总修正 |
| `pwm` | 最终左右 PWM |

### 3.4 B 点

CB 成功后：

- 正常停止原因是 `reason=arc_done`
- 触发一次 ST011 声光
- 读取 `B_exit` 仅用于日志分析
- BD 固定目标角为 `RACE_BD_HEADING_TARGET_CDEG`

### 3.5 BD 直线

- **目标航向**：`RACE_BD_HEADING_TARGET_CDEG`（-10638）
- **控制方式**：航向直线
- **结束条件**：扫到 D 点线

**判断方式**：

| 现象 | 说明 |
|------|------|
| `h_raw` 能接近 `h_tgt`，但 `reason=force` | 航向大体到了，D 点线没扫到 |
| `h_corr/corr` 长期贴限幅 | 目标角过远或转向能力不足 |
| `raw/mask/cnt` 曾经有线但没停 | 入线判定或 arm 距离需要检查 |
| `raw=0xFF mask=0x00` 持续到 force | 红外一直没看到 D 点线 |

### 3.6 DA 弧线

DA 是右弧线，同样使用竞速弧线控制：

- `dir=1`
- `stop_on_final_line=1`
- 到最终 A 点时，红外点位条件触发 `reason=line`
- 成功后刹车、声光提示并重置编码器距离计数

## 5. 常见问题判断

### 5.1 C 点扫不到线

看 `TASK3_AC stop`：

- `reason=line`：AC 成功扫到 C 点
- `reason=force`：没扫到线，优先看 `h_tgt/h_raw`、`search` 和 `raw/mask/cnt`

如果车身姿态接近但红外没看到线，优先调 AC 目标角或寻线距离；如果车身姿态明显不对，优先调 `TASK3_AC_HEADING_TARGET_CDEG`。

### 5.2 CB 弧线丢线

看 `TASK3_CB`：

- `valid=0` 或 `seen=0`：红外当前没有有效线
- `line_turn` 长期很大：红外误差大
- `nav_turn` 长期同向很大：JY62 弧线模型认为车身角度落后或超前
- `tdiff=-44`：左弧线差速目标正在生效

先判断是红外没跟住，还是 JY62 模型在持续拉偏。

### 5.3 BD 到不了 D

当前 BD 是固定目标角。看：

```text
TASK3_BD target: B_exit=... target=-13920 mode=absolute
TASK3_BD t=... h_raw=... h_tgt=...
```

如果多次都偏同一边，调 `TASK3_BD_HEADING_TARGET_CDEG`。

### 5.4 DA 最后 A 点停得晚

看 `TASK3_DA stop`：

- 期望：`reason=line`
- 如果 `reason=force`，说明 A 点出线没有被成功识别
- 如果出了线又走一段才停，检查 `RACE_ARC_POINT_YAW_ARM_CDEG`、弧线点位 arm 距离和最终线判定

---

## 6. 调参顺序建议

1. 先跑任务一，确认基础直线和编码器正常
2. 跑任务三，先看 AC 是否稳定扫到 C
3. AC 成功后看 CB 是否稳定 `reason=arc_done`
4. 再调 BD 固定目标角，确保扫到 D
5. 最后调 DA 的 A 点最终停车
6. 每次只改一个参数，并保留对应串口日志
