# 任务三调试说明

本文记录任务三当前实现逻辑、关键参数和串口字段。任务三入口为 `B24` 按键或 UART0 `03`。

## 1. 路线目标

任务三按以下顺序运行：

```text
A 点出发，车头朝 C
TASK3_AC: A -> C 直线
TASK3_CB: C -> B 弧线红外循迹
TASK3_BD: B -> D 直线，按 B 点实际出线航向相对转向
TASK3_DA: D -> A 弧线红外循迹
A 点出线停车
```

任务三不复用任务二弧线逻辑。CB 和 DA 使用 `run_task3_arc_line_follow_segment()`，便于单独调任务三弧线入口、出线和丢线策略。

## 2. 分段逻辑

### 2.1 TASK3_AC

`TASK3_AC` 复用直线段函数 `run_straight_to_line_segment()`：

- 启动时触发声光。
- 延时后把 JY62 当前航向置零。
- 使用编码器速度 PID、累计距离修正、JY62 航向修正保持直线。
- 到 `TASK3_STRAIGHT_LINE_ARM_COUNT` 后允许红外入线触发。
- AC 入线要求中间区域命中：`TASK3_AC_STOP_CENTER_MASK`，并要求 `active_count >= TASK3_AC_STOP_MIN_IR_COUNT`。

### 2.2 C 点

AC 扫到 C 点入线后：

- 触发一次 ST011 声光。
- 不使用任务二弧线，进入任务三专用 CB 弧线。

### 2.3 TASK3_CB

CB 弧线由红外循迹主导：

- `entry=1` 时处于入弯强化区，基础 PWM 降低，并叠加 `TASK3_ARC_ENTRY_TURN`。
- `valid=1` 时使用红外误差 `err/filt/der` 计算转向。
- `valid=0` 时使用丢线补偿转向。
- `yaw_progress` 根据 JY62 相对航向累计弧线进度。
- 当到达 B 点附近角度，并且红外出线时，结束 CB，停止原因是 `reason=arc_done`。

B 点出线判断不是只看距离。当前重点字段：

| 字段 | 含义 |
|---|---|
| `yrdy` | 弧线累计角度是否到达 `TASK3_ARC_YAW_DONE_CDEG` |
| `brdy` | 当前航向是否接近 B 点理论出线方向 |
| `bmark` | 曾经满足 B 点角度条件后的锁存标志 |
| `xcnt` | 出线确认计数 |
| `reason=arc_done` | CB 成功从 B 点出线 |

### 2.4 B 点

CB 成功出线后：

- 触发一次 ST011 声光。
- 不刹车。
- 直接进入 BD，保持“边走边转”。

BD 目标角计算方式：

```c
b_exit_heading = nav.yaw_relative_cdeg;
bd_heading_target = normalize_cdeg(b_exit_heading + TASK3_BD_RELATIVE_TURN_CDEG);
```

当前：

```c
#define TASK3_BD_RELATIVE_TURN_CDEG (6700)
```

也就是以 B 点实际出线姿态为基准，相对左转 67.00 度。

### 2.5 TASK3_BD

BD 复用直线段函数，但传入目标航向并开启 `heading_only`：

- 编码器速度 PID 和累计距离修正会被压掉。
- 主要由 JY62 航向目标控制转向。
- 到 `TASK3_STRAIGHT_LINE_ARM_COUNT` 后开启红外寻线。
- 扫到 D 点线后进入 DA。
- 如果没扫到线，到 `TASK1_FORCE_STOP_COUNT` 会 `reason=force` 中止。

关键串口行：

```text
TASK3_BD target: B_exit=... B_nom=18000 rel_turn=6700 target=... mode=relative
TASK3_BD t=... h_raw=... h_tgt=... h_corr=... corr=... B_pwm=... A_pwm=...
TASK3_BD stop: reason=line/force ...
```

判断方式：

| 现象 | 说明 |
|---|---|
| `h_raw` 很快接近 `h_tgt`，但 `reason=force` | 航向到了，但没扫到 D 点线，调相对角或寻线 |
| `h_corr/corr` 长期打满 | 航向目标太远或转向能力不足 |
| `reason=line` | BD 成功扫到 D 点线 |
| `raw=0xFF mask=0x00` 持续到 force | 红外一直没看到 D 点线 |

### 2.6 TASK3_DA

DA 使用任务三专用红外弧线，方向与 CB 相反：

- 由红外误差循迹。
- 最终 A 点需要到达出线角度窗口后，由红外出线或宽线判定结束。
- 成功时 `reason=line`，随后触发结束声光并刹车。

## 3. TASK3_CB / TASK3_DA 字段

弧线周期日志示例：

```text
TASK3_CB t=1500 dist=6790 yaw=17090 ystep=186 yprog=17555 nav=1 ir=1 valid=1 wide=0 final=0 yrdy=1 drdy=0 frdy=0 brdy=1 bmark=1 xcnt=0 seen=1 lost_cnt=0 entry=0 mode=0 raw=0xCF mask=0x0C cnt=2 err=-1000 filt=-999 der=0 turn=-90 L=340 R=550
```

| 字段 | 含义 |
|---|---|
| `dist` | 当前弧线段编码器距离估计 |
| `yaw` | 当前 JY62 相对航向，单位 0.01 度 |
| `ystep` | 本周期对弧线方向有效的航向增量 |
| `yprog` | 弧线累计航向进度 |
| `valid` | 红外当前是否有效看到线 |
| `wide` | 是否看到宽线或多探头大面积触发 |
| `final` | DA 最终 A 点出线判定 |
| `yrdy` | 角度进度是否满足弧线结束门槛 |
| `drdy` | 距离是否满足弧线结束门槛 |
| `frdy` | DA 最终出线是否已允许 |
| `brdy` | CB 是否接近 B 点出线方向 |
| `bmark` | CB 中 B 点出线方向锁存 |
| `xcnt` | 出线确认计数 |
| `seen` | 本段是否曾经看到过有效线 |
| `lost_cnt` | 连续丢线次数 |
| `entry` | 是否在入弯强化区 |
| `mode` | 弧线控制模式：0 正常，1 入弯丢线，2 入弯见线，3 中后段丢线 |
| `err/filt/der` | 红外误差、滤波误差、差分项 |
| `turn` | 最终转向量 |
| `L/R` | 左右轮 PWM |

## 4. 常见问题判断

### B 点没有响

看 `TASK3_CB stop`：

- 正常：`reason=arc_done bmark=1 xcnt>=1`
- 异常：`reason=force` 或 `reason=timeout`

如果是 `reason=force`，说明没有判到 B 点红外出线。优先看 `bmark` 是否曾经为 1，以及出线时 `raw/mask/cnt` 是否符合。

### B 点出线后没有向 D 点转够

看 `TASK3_BD target`：

- `B_exit` 是 B 点实际出线航向。
- `rel_turn` 是相对转角。
- `target` 是 BD 目标航向。

如果 `h_raw` 很快到 `h_tgt`，但车实际没到 D，调 `TASK3_BD_RELATIVE_TURN_CDEG`。

### BD 扫不到 D 点线

看 `TASK3_BD stop`：

- `reason=force`：没有扫到线。
- `raw=0xFF mask=0x00 cnt=0`：红外始终没看到线。
- `raw/mask/cnt` 曾经有线但没停：入线判定过严或 `arm` 还没打开。

### DA 最后 A 点停得晚

看 `TASK3_DA stop`：

- 正常应由 `reason=line` 结束。
- 如果车出线后又走一段才响，说明 DA 的 `final_exit_ready` 或最终线判定还需要收紧。

## 5. 调参顺序建议

1. 先让 CB 稳定 `reason=arc_done`。
2. 再调 `TASK3_BD_RELATIVE_TURN_CDEG`，保证 BD 能到 D 点附近。
3. 如果 BD 到了附近但扫不到线，再调 BD 寻线距离和摆头强度。
4. 最后调 DA 末端 A 点出线停车。
5. 每次只改一个参数，并保留对应串口日志。
