# 八路红外循迹模块驱动说明

本文档对应当前工程中的：

- `BSP/inc/bsp_ir_tracking.h`
- `BSP/src/bsp_ir_tracking.c`
- `main.c` 中的 `run_ir_tracking_uart_test()`
- `main.syscfg` 中的 `I2C_0`

当前目标不是直接让小车循迹跑起来，而是先确认八路红外模块能稳定地通过 I2C 被 MSPM0 读取，并通过串口打印出可理解的数据。实测结果已经证明当前 I2C 通信、黑白逻辑和左右方向判断是正常的。

## 1. 当前接线

| 模块信号 | MSPM0G3507 | SysConfig 名称 | 说明 |
|---|---|---|---|
| `SDA` | `PB3 / B03` | `I2C_0_SDA` | I2C 数据线 |
| `SCL` | `PB2 / B02` | `I2C_0_SCL` | I2C 时钟线 |
| `VCC` | 优先 `3V3` | - | 避免 I2C 上拉到 5V |
| `GND` | `GND` | - | 必须和主控共地 |

当前 `Debug/ti_msp_dl_config.h` 中可以看到：

```c
#define I2C_0_INST I2C1
#define I2C_0_BUS_SPEED_HZ 400000
```

也就是说 SysConfig 实际使用的是 MSPM0 的 `I2C1` 外设，但工程里给它起名为 `I2C_0`。写代码时使用 `I2C_0_INST` 即可，不需要手动关心底层实例名。

## 2. 模块协议

商家例程中的 I2C 协议为：

| 项目 | 值 |
|---|---|
| I2C 设备地址 | `0x12` |
| 数字量寄存器 | `0x30` |
| 数据长度 | 1 字节 |

读取 `0x30` 后得到的原始字节含义：

```text
bit7 bit6 bit5 bit4 bit3 bit2 bit1 bit0
 X1   X2   X3   X4   X5   X6   X7   X8
```

当前模块实测逻辑：

| 原始位值 | 含义 |
|---|---|
| `0` | 该路检测到黑线 |
| `1` | 该路检测到白底 |

## 3. raw 和 mask 的区别

驱动保留两个值：

| 字段 | 含义 |
|---|---|
| `raw` | 模块原始字节，`bit7=X1`，`bit0=X8` |
| `line_mask` | 程序转换后的黑线掩码，`bit0=X1`，`bit7=X8`，1 表示黑线 |

这样做是为了让后续控制代码更直观。控制代码只需要从左到右看 `line_mask` 的 bit0 到 bit7，就能知道 X1 到 X8 哪些探头压到了黑线。

举例：

```text
raw  = 0xE7 = 1110 0111
mask = 0x18 = 0001 1000
```

解释：

- `raw` 中 bit4 和 bit3 为 0，说明 X4、X5 检测到黑线。
- 转换后 `mask` 中 bit3 和 bit4 为 1，也表示 X4、X5 检测到黑线。
- X4、X5 位于中间，所以位置误差为 0。

## 4. 位置误差计算

驱动给 X1 到 X8 设置了从左到右的权重：

```text
X1    X2    X3    X4    X5    X6    X7    X8
-3500 -2500 -1500 -500  +500  +1500 +2500 +3500
```

计算方式：

```text
position = 所有压线探头权重之和 / 压线探头数量
error    = position
```

含义：

| `error` | 含义 |
|---|---|
| 负数 | 黑线在传感器左侧 |
| 0 附近 | 黑线在传感器中心 |
| 正数 | 黑线在传感器右侧 |

如果 8 路都没有检测到黑线，`line_lost` 会变成 1。此时驱动不会把误差清零，而是返回最近一次有效误差，方便后续小车按照最后偏离方向低速找线。

## 5. 主要函数

### IRTracking_Init()

初始化驱动内部状态：

```c
IRTracking_Init();
```

注意：I2C 外设本身由 `SYSCFG_DL_init()` 初始化，所以主程序顺序应为：

```c
SYSCFG_DL_init();
IRTracking_Init();
```

### IRTracking_ReadRaw()

读取模块原始字节：

```c
uint8_t raw;

if (IRTracking_ReadRaw(&raw) != 0U) {
    lc_printf("raw=0x%02X\r\n", raw);
}
```

返回值：

| 返回值 | 含义 |
|---|---|
| `1` | 读取成功 |
| `0` | 读取失败 |

读取失败常见原因：

- 模块没有供电。
- `SDA` 和 `SCL` 接反。
- `GND` 没有共地。
- I2C 地址不是 `0x12`。
- 模块 I2C 上拉到了 5V，导致 MSPM0 侧电平不匹配。

### IRTracking_RawToLineMask()

把模块原始字节转换成黑线掩码：

```c
uint8_t mask = IRTracking_RawToLineMask(raw);
```

正常主程序一般不需要单独调用它，因为 `IRTracking_ReadSample()` 已经包含了这一步。

### IRTracking_ReadSample()

读取并解析一次完整循迹状态：

```c
ir_tracking_sample_t sample;

if (IRTracking_ReadSample(&sample) != 0U) {
    IRTracking_PrintSample(&sample);
}
```

`ir_tracking_sample_t` 字段说明：

| 字段 | 含义 |
|---|---|
| `raw` | 模块原始字节 |
| `line_mask` | 黑线掩码 |
| `active_count` | 当前检测到黑线的探头数量 |
| `line_lost` | 是否丢线 |
| `position` | 加权线位置 |
| `error` | 给循迹控制器使用的误差 |

### IRTracking_PrintSample()

打印采样结果：

```c
IRTracking_PrintSample(&sample);
```

输出示例：

```text
IR raw=0xE7 mask=0x18 count=2 lost=0 pos=0 err=0
```

## 6. 当前测试模式

`main.c` 中有红外模块串口测试开关：

```c
#define ENABLE_IR_TRACKING_UART_TEST (1)
```

当它为 1 时：

- 程序会初始化 SysConfig、编码器状态和 TB6612。
- TB6612 会保持刹车。
- 程序进入 `run_ir_tracking_uart_test()`。
- 每 `100ms` 读取一次八路红外模块，并通过 UART0 打印。
- 不会进入直行 PID，电机不会主动转动。

测试完成后，如果要恢复跑车，把它改成：

```c
#define ENABLE_IR_TRACKING_UART_TEST (0)
```

然后重新编译烧录。

## 7. 实测结果参考

下面是已经在场地图上测到的典型结果，可以作为以后排错参照。

### 黑线在传感器中间

```text

```

说明 X4、X5 压线，黑线居中，方向正确。

### 黑线向左移动

```text

IR raw=0xDF mask=0x04 count=1 lost=0 pos=-1500 err=-1500
IR raw=0xBF mask=0x02 count=1 lost=0 pos=-2500 err=-2500
IR raw=0x7F mask=0x01 count=1 lost=0 pos=-3500 err=-3500
```

说明黑线越靠左，误差越负。


说明黑线越靠右，误差越正。

## 8. 后续接入循迹控制

后续可以把红外误差接入一个循迹 PID：

```text
红外 error -> 循迹 PID -> turn
基础速度 base_pwm -> 左右轮 PWM
编码器速度差 -> 小范围补偿左右轮速度
```

一个最简单的差速思路是：

```c
left_pwm  = base_pwm + turn;
right_pwm = base_pwm - turn;
TB6612_SetDifferential(left_pwm, right_pwm);
```

这里的正负号必须用实车确认。当前驱动约定：

- `TB6612_SetDifferential(left, right)` 的第一个参数是左轮 B 电机。
- 第二个参数是右轮 A 电机。
- 红外 `error` 为负表示线在左侧，为正表示线在右侧。

## 9. CCS 工程注意事项

本次新增的源文件是：

```text
BSP/src/bsp_ir_tracking.c
```

如果编译时出现 `IRTracking_ReadSample`、`IRTracking_PrintSample` 等未定义符号，通常不是函数写错，而是 CCS 工程还没有把新 `.c` 文件纳入构建。处理方式：

1. 在 CCS 里右键工程，选择 `Refresh`。
2. 执行 `Clean Project`。
3. 再执行 `Build Project`。

不要手动修改 `Debug/subdir_vars.mk`、`Debug/makefile` 等自动生成文件。
