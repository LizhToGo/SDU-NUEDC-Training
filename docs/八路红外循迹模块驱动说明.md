# 八路红外循迹模块驱动说明

八路红外模块通过 I2C 读取，当前用于：

- 任务一/二终点线检测。
- 任务三/四 AC/BD 点位检测。
- 任务三/四 CB/DA 弧线巡线。
- 串口诊断中输出线位置、丢线和误差。

当前验收版不再保留独立红外串口测试入口，也没有 `ENABLE_IR_TRACKING_UART_TEST` 或纯红外循迹调试模式。

## 接口

主要接口在 BSP 红外驱动中：

```c
IRTracking_Init();
IRTracking_ReadSample(&sample);
```

常用字段：

| 字段 | 含义 |
|---|---|
| `raw` | 原始 8 位传感器值 |
| `line_mask` | 黑线命中掩码 |
| `active_count` | 命中数量 |
| `line_lost` | 是否丢线 |
| `error` | 线相对车身中心的误差 |

## mask 与误差

示例：

```text
raw  = 0xE7 = 1110 0111
mask = 0x18 = 0001 1000
```

中间传感器命中通常表示线在车身中心附近；线偏到左/右侧时，`error` 会随之变大。具体正负方向以当前驱动和日志为准，实车复盘时结合车身偏移观察。

## 在任务中的作用

任务一/二：

- 红外用于终点线检测。
- 直线主要由编码器差速和 JY62 航向修正控制。

任务三/四：

- AC/BD 直线段红外主要用于点位检测和保护。
- CB/DA 弧线段红外参与巡线。
- 强制入弯后会通过红外找线完成交接。

## 日志字段

| 字段 | 含义 |
|---|---|
| `raw` | 原始值 |
| `mask` | 命中掩码 |
| `cnt` / `active_count` | 命中数量 |
| `lost` / `line_lost` | 是否丢线 |
| `err` / `line_error` | 红外误差 |

如果日志中红外扫到线但没有触发入弯，先检查当前距离是否已经超过对应 `point_arm_count`。未到 arm 距离时，点位线会被保护逻辑忽略。
