# 项目实现思路

本文从软件结构、硬件抽象、控制算法和调试流程说明当前自动行驶小车固件。当前工程基于 TI MSPM0G3507，使用 CCS、SysConfig 和 DriverLib。

## 1. 总体目标

当前固件的目标是把电机、编码器、八路红外循迹、JY62 航向和 UART0 调试整合成一套可反复实车调参的任务框架。

核心思路：

- 直线段依靠编码器速度闭环、左右累计距离修正和 JY62 航向修正。
- 到点依靠红外模块识别黑线，并用编码器距离作为兜底。
- 弧线段依靠红外循迹，JY62 和编码器用于判断弧线进度和出线窗口。
- 任务状态机负责 A/B/C/D 分段切换。
- 串口日志尽量打印每一层控制量，便于根据实车现象定位原因。
- 参数集中放在 `app_config.h`，减少调参时改动控制逻辑的风险。

## 2. 软件分层

| 层级 | 文件 | 作用 |
|---|---|---|
| SysConfig 配置层 | `main.syscfg` | 时钟、GPIO、PWM、UART、I2C、中断等外设配置源头 |
| BSP 驱动层 | `BSP/inc`、`BSP/src` | TB6612、编码器、八路红外、JY62 等硬件接口 |
| 控制辅助层 | `app_control.h`、`app_config.h` | 参数、限幅、斜坡、PID、航向滤波 |
| 任务应用层 | `main.c`、`app_debug_modes.h` | 任务调度、直线段、弧线段、调试模式、串口输出 |

当前 `bsp_encoder.h` 和 `bsp_jy62.h` 采用头文件内 `static` 实现，只在 `main.c` 中 include 一次。这样做是为了避免当前 CCS 自动生成 Makefile 没有纳入新增 `.c` 文件时影响编译。

## 3. 硬件抽象

### 3.1 电机和车轮映射

实车约定：

| TB6612 通道 | 实际车轮 | PWM |
|---|---|---|
| A 电机 | 右轮 | `PA0 / PWMA` |
| B 电机 | 左轮 | `PA1 / PWMB` |

上层统一使用车体语义：

```c
TB6612_SetDifferential(left_speed, right_speed);
```

### 3.2 编码器

| 车轮 | 编码器引脚 |
|---|---|
| 左轮 B | `PA14 / PA15` |
| 右轮 A | `PA16 / PA17` |

编码器提供两类数据：

- 周期增量：用于估算 `B_spd/A_spd`。
- 当前段累计量：用于估算 `dist` 和 `d_err`。

每段任务开始时调用 `encoder_reset_distance_counts()`，避免上一段累计量影响下一段。

### 3.3 八路红外循迹模块

红外模块通过 I2C 读取数字量，驱动层输出：

- `raw`：模块原始值。
- `mask`：黑线掩码，`bit0..bit7` 对应 `X1..X8`。
- `active_count`：看到黑线的探头数量。
- `error`：线相对传感器中心的位置误差。
- `line_lost`：是否丢线。

直线段主要用红外判断到点；弧线段用红外误差做循迹控制。

### 3.4 JY62 航向

JY62 使用 UART1 接收姿态帧。驱动解析角度帧和角速度帧，并提供：

- 当前 yaw：`yaw_cdeg`。
- 相对任务开始置零后的 yaw：`yaw_relative_cdeg`。
- Z 轴角速度：`gyro_z_mdps`。
- 低通 Z 轴角速度：`gyro_z_filtered_mdps`。
- 有效标志：`valid`。

任务开始时可调用 `JY62_SetYawZeroToCurrent()`，把当前车头方向作为 0 航向。

## 4. 任务调度

正式模式由 `run_task_dispatcher()` 负责等待按键或 UART0 命令。

| 命令 | 功能 |
|---|---|
| `01` | 任务一 A -> B |
| `02` | 任务二 A -> B -> C -> D -> A |
| `03` | 任务三 A -> C -> B -> D -> A |
| `04` | 任务四入口，占位 |
| `05` | 轮速 / 编码器 / PWM 测试 |
| `00` | 强制停车 |

实体按键与 UART0 对应：`A26=01`、`A24=02`、`B24=03`、`A22=04`。

## 5. 直线段控制

直线段由 `run_straight_to_line_segment()` 实现，任务一、任务二直线段、任务三 AC/BD 都复用这一套逻辑。

### 5.1 起步斜坡

直线段从起步 PWM 逐步过渡到目标 PWM：

```c
#define TASK1_RAMP_B_START_PWM    (500)
#define TASK1_RAMP_A_START_PWM    (540)
#define STRAIGHT_B_BASE_PWM       (532)
#define STRAIGHT_A_BASE_PWM       (543)
```

### 5.2 速度 PID

速度 PID 的目标是让左右轮当前周期速度差接近目标值：

```text
v_err = (B_spd - A_spd) - target_diff
```

直线段目标一般为 0。

### 5.3 累计距离修正

累计距离误差：

```text
d_err = B_total - A_total
```

`d_corr` 用于抑制长距离慢慢偏航。

### 5.4 JY62 航向修正

JY62 航向误差会经过低通、死区和晃动降权，再转换成 `h_corr`。最终修正量近似为：

```text
corr = v_corr + d_corr + h_corr + search
```

再作用到左右轮：

```text
B_pwm = base_b_pwm - corr
A_pwm = base_a_pwm + corr
```

BD 这类指定目标航向直线段会使用 `heading_only=1`，此时主要由 `h_corr` 控制方向。

## 6. 任务二弧线

任务二弧线由 `run_arc_line_follow_segment()` 实现，主要特点：

- 根据弧线半径和轮距计算左右轮目标速度差。
- 使用编码器差速 PID 作为主控制。
- 红外误差和 JY62 弧线进度作为修正。
- 通过 `yaw_progress`、出线窗口和强制距离判断结束。

任务二目前继续保留这套逻辑，和任务三弧线分开调。

## 7. 任务三弧线

任务三弧线由 `run_task3_arc_line_follow_segment()` 实现，独立于任务二。

当前特点：

- 入弯初期 `entry=1`，降低基础速度并叠加较大的入弯转向。
- 正常见线时用红外误差 `err/filt/der` 计算转向。
- 丢线时按最近转向或弧线方向进行补偿。
- CB 出 B 点时要求角度进度、B 点方向锁存和红外出线共同成立。
- DA 最终 A 点要求进入最终出线窗口后，由红外出线结束。

任务三 B->D 的目标角使用相对公式：

```text
BD target = B_exit_heading + TASK3_BD_RELATIVE_TURN_CDEG
```

这样 B 点出线姿态有轻微偏差时，BD 仍按实际姿态继续边走边转。

## 8. 调试模式

| 模式 | 用途 |
|---|---|
| UART0 `05` | 单独观察编码器、轮速、PID 和 PWM |
| 红外 UART 测试 | 单独查看八路红外 `raw/mask/cnt/error` |
| 纯红外循迹测试 | 单独测试红外循迹转向 |
| 编码器自检 | 架空车体时检查左右编码器是否有脉冲 |

调车时优先拆分问题：

- 直线偏：先跑 `05` 看轮速和编码器。
- 识别不到线：先跑红外测试看 `raw/mask/cnt`。
- 航向异常：看 `JY62 ok/frames/rel_cdeg/gzlp_mdps`。
- 弧线失控：看弧线段 `valid/lost_cnt/err/turn/L/R`。

## 9. 安全和兜底策略

当前固件保留：

- UART0 `00` 强制停车。
- 直线段最大运行时间和强制距离停车。
- 弧线段最大运行时间和强制距离停车。
- 每段失败打印 `abort after ...`，避免静默进入下一段。
- 声光模块大多采用非阻塞触发，减少段切换等待。

总体原则：先保证每一段可解释，再追求速度。只要日志能解释实车动作，后续调参就不会乱。
