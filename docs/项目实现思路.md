# 项目实现思路

本文从软件结构、硬件抽象、控制算法和调试流程说明当前自动行驶小车固件。当前工程基于 TI MSPM0G3507，使用 CCS、SysConfig 和 DriverLib。

## 1. 总体目标

当前固件的目标是把电机、编码器、八路红外循迹、JY62 航向、ST011 声光和 UART0 调试整合成一套可反复实车调参的任务框架。

核心思路：

- 直线段用编码器差速闭环和 JY62 航向修正。
- 到点依靠红外模块识别黑线，并用编码器距离作为兜底。
- 弧线段用红外 PD、轮速差目标和 JY62 弧线模型共同修正。
- 任务状态机负责 A/B/C/D 分段切换。
- 参数集中放在 `app_config.h`，减少调参时改动控制逻辑的风险。
- Task11 作为经验采集平台，用全程黑线赛道验证任务三/四未来可复用的距离和航向参数。

## 2. 软件分层

| 层级 | 文件 | 作用 |
|---|---|---|
| SysConfig 配置层 | `main.syscfg` | 时钟、GPIO、PWM、UART、I2C、中断等外设配置源头 |
| BSP 驱动层 | `BSP/inc`、`BSP/src` | TB6612、编码器、八路红外、JY62 等硬件接口 |
| 控制辅助层 | `app_control.h` | 限幅、斜坡、直线 PID、航向滤波 |
| 参数层 | `app_config.h` | PWM、PID、距离门槛、角度目标、任务开关 |
| 差速控制层 | `app_straight.h` | 将轮速差目标、前馈、反馈、累计距离差转换为左右 PWM |
| 调试模式层 | `app_debug_modes.h` | 05/07 轮速测试、红外打印、纯循迹测试 |
| 任务应用层 | `main.c` | 任务调度、任务一二三四、任务六、任务十/十一 |
| 日志工具 | `tools/task11_log_to_csv.py` | Task11 串口文本转 CSV 和 summary |

当前 `bsp_encoder.h` 和 `bsp_jy62.h` 采用头文件内 `static` 实现，只在 `main.c` 中 include 一次。这样做是为了避免当前 CCS 自动生成 Makefile 没有纳入新增 `.c` 文件时影响编译。

## 3. 硬件抽象

### 3.1 电机和车轮映射

实车约定：

| TB6612 通道 | 实际车轮 | PWM |
|---|---|---|
| A 电机 | 右轮 | `PA0 / PWMA` |
| B 电机 | 左轮 | `PA1 / PWMB` |

上层统一使用车体语义：

```c
TB6612_SetDifferential(left_speed, right_speed);
```

### 3.2 编码器

| 车轮 | 编码器引脚 |
|---|---|
| 左轮 B | `PA14 / PA15` |
| 右轮 A | `PA16 / PA17` |

编码器提供两类数据：

- 周期增量：用于估算 `B_spd/A_spd`。
- 当前段累计量：用于估算 `dist`、`d_err` 和任务点距离。

每段任务开始时会重置当前段距离计数，避免上一段累计量影响下一段。

### 3.3 八路红外循迹模块

红外模块通过 I2C 读取数字量，驱动层输出：

- `raw`：模块原始值。
- `mask`：黑线掩码，`bit0..bit7` 对应 `X1..X8`。
- `active_count`：看到黑线的探头数量。
- `error`：线相对传感器中心的位置误差。
- `line_lost`：是否丢线。

直线段主要用红外判断到点；Task11 和弧线段还会用红外误差做循迹控制。

### 3.4 JY62 航向

JY62 使用 UART1 接收姿态帧。驱动解析角度帧和角速度帧，并提供：

- 当前 yaw：`yaw_cdeg`。
- 相对最近一次置零后的 yaw：`yaw_relative_cdeg`。
- Z 轴角速度：`gyro_z_mdps`。
- 低通 Z 轴角速度：`gyro_z_filtered_mdps`。
- 更新标志：`update_flags`。
- 有效标志：`valid`。

Task10 和部分任务启动时会调用 `JY62_SetYawZeroToCurrent()`，把当前车头方向作为相对 0 航向。

## 4. 任务调度

正式模式由 `run_task_dispatcher()` 等待按键或 UART0 命令。

| 命令 | 功能 |
|---|---|
| `01` | 任务一 A -> B |
| `02` | 任务二 A -> B -> C -> D -> A |
| `03` | 任务三 A -> C -> B -> D -> A |
| `04` | 任务四，按任务三路线连续 4 圈 |
| `05` | PID 轮速 / 编码器 / PWM 测试 |
| `06` | 任务六，A->C 后 C 点快速左转测试 |
| `07` | PD 轮速 / 编码器 / PWM 测试 |
| `10` | Task10，AB 方向零航向标定 |
| `11` | Task11，全程黑线经验采集 |
| `00` | 强制停车 |

实体按键与正式任务对应：`A26=01`、`A24=02`、`B24=03`、`A22=04`。

## 5. 直线段控制

当前代码中有两套直线段实现：

| 函数 | 主要用途 |
|---|---|
| `run_straight_to_line_segment()` | 任务一、任务二 AB、任务二 CD 的差速直线控制 |
| `run_heading_straight_to_line_segment()` | 任务三/四 AC、BD，带寻线保护和更强目标航向控制 |

### 5.1 差速直线模块

`app_straight.h` 中的 `straight_drive_update()` 是当前复用最多的基础模块。它输入：

- 左右轮周期增量。
- 左右轮累计计数。
- 基础 PWM。
- 目标速度差。
- 前馈增益、PID 限幅、累计距离修正参数。

输出：

- `feedforward_correction`
- `feedback_correction`
- `distance_correction`
- `correction`
- `motor_b_pwm/motor_a_pwm`

修正约定：

```text
correction > 0：降低左轮 B，提高右轮 A
correction < 0：提高左轮 B，降低右轮 A
```

### 5.2 JY62 航向修正

直线段会根据目标航向计算航向误差，再结合低通角速度做阻尼。任务一风格直线使用 `task5_yaw_correction()`；任务三/四 AC、BD 使用 `run_heading_straight_to_line_segment()` 内部的目标航向控制和寻线保护。

当前任务三/四的直线目标不是相对转角，而是固定绝对航向：

```text
AC target = TASK3_AC_HEADING_TARGET_CDEG
BD target = TASK3_BD_HEADING_TARGET_CDEG
```

## 6. 弧线段控制

当前保留的弧线主流程如下，早期不可达弧线函数已在 2026-06-10 清理：

| 函数 | 状态 |
|---|---|
| `run_task2_task11_arc_segment()` | 任务二当前调试弧线 |
| `run_task3_task11_arc_line_follow_segment()` | 任务三/四当前主要弧线 |
| `run_task11_ir_map_test_laps()` | 任务三/四当前入口使用的 Task11 风格多圈流程 |

Task11 风格弧线控制大致为：

1. 根据当前路段设置目标轮速差，例如 CB 为负、DA 为正。
2. 用 `straight_drive_update()` 输出基础左右 PWM。
3. 红外有效时用 `err/filt/der` 计算 `line_turn`。
4. JY62 有效时根据弧线理论角度计算 `nav_turn`。
5. 最终 `turn = line_turn + nav_turn`，再叠加到左右轮 PWM。
6. 点位判断由距离、JY62 角度门槛和红外边缘/宽线共同决定。

## 7. 任务三/四策略

任务三：

```text
TASK3_AC: 固定航向直线到 C
TASK3_CB: Task11 风格左弧线到 B
TASK3_BD: 固定航向直线到 D
TASK3_DA: Task11 风格右弧线到 A 并停车
```

任务四：

```text
按任务三四段连续执行 4 圈
```

前三圈 A 点出线后继续下一圈，第四圈 A 点停车。当前不再根据 A/B 点出线瞬间航向动态计算下一段目标角。

## 8. Task10 / Task11 经验采集

Task10：

- 车头朝 AB 摆放。
- 发送 `10`。
- JY62 相对航向清零。
- 当前任务三/四的 AC、BD 固定目标角默认以这个 AB 零航向为基准。

Task11：

- 车头从 A 指向 C。
- AC 和 BD 都贴黑线，全程可被红外模块看到。
- 运行 5 圈。
- 记录点位距离、航向、红外 mask、轮速差、快转前后角度。
- 数据可用 `tools/task11_log_to_csv.py` 转成 CSV。

当前默认关闭 Task11 大体量日志。如需采集，先在 `app_config.h` 打开 `TASK11_RAM_LOG_ENABLE` 或 `TASK11_UART_LOG_ENABLE`。

## 9. 安全和兜底策略

当前固件保留：

- UART0 `00` 强制停车。
- 直线段最大运行时间和强制距离停车。
- 弧线段最大运行时间和强制距离停车。
- 每段失败打印 `abort after ...`，避免静默进入下一段。
- ST011 声光大多采用非阻塞 pulse，减少段切换等待。

总体原则：先保证每一段可解释，再追求速度。只要日志能解释实车动作，后续调参就不会乱。
