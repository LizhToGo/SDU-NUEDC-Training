## 1. 当前可用状态

- 当前正式任务调度已启用：`ENABLE_CONTEST_TASKS=1`。

- UART0 命令：
  - `01`：任务一 A -> B 直线。
  - `02`：任务二 A -> B -> C -> D -> A。
  - `03`：任务三 A -> C -> B -> D -> A。
  - `04`：任务四入口，占位。
  - `05`：左右轮编码器/PWM 测试。
  - `00`：强制停车。
- 实体任务按键：
  - `A26 / PA26`：任务一。
  - `A24 / PA24`：任务二。
  - `B24 / PB24`：任务三。
  - `A22 / PA22`：任务四。
- 任务一当前直线稳定，可作为基础直线留档。
- 任务二可运行，仍需继续看弧线入口和出线姿态。

## 3. 任务三当前逻辑

任务三入口：`B24` 或 UART0 `03`。

流程：

1. `TASK3_AC`：A -> C 直线。启动时 JY62 置零，直线控制逻辑和任务一相同，接近 C 点后开启红外寻线。
2. C 点：AC 直线扫到入线后触发声光，进入 CB 弧线。
3. `TASK3_CB`：任务三专用红外弧线。入弯初期给更强的转向辅助，随后由红外误差闭环循迹。
4. B 点：CB 段中，当航向进度和 B 点出线条件成立，并且红外判定出线时，`reason=arc_done`。此时触发声光，不刹车，直接进入 BD。
5. `TASK3_BD`：B -> D 直线。目标航向不是固定角，而是 `B_exit + TASK3_BD_RELATIVE_TURN_CDEG`，当前相对左转 67 度。
6. D 点：BD 直线扫到线后触发声光，进入 DA 弧线。
7. `TASK3_DA`：任务三专用红外弧线。最终 A 点由红外出线触发停车和结束声光。

任务三详细字段见 [任务三调试说明.md](任务三调试说明.md)。

## 4. 当前已知问题

### 4.1 任务三 B -> D 仍需调角度

最近几次现象：

- B 点出线后已经能进入 `TASK3_BD`。
- 之前 `rel_turn=3800/5000` 时左转角偏小，容易错过 D 点。
- 当前留档参数已经提到 `6700`，需要继续实车验证是否能稳定扫到 D 点线。

看日志时重点关注：

```text
TASK3_BD target: B_exit=... rel_turn=6700 target=... mode=relative
TASK3_BD t=... h_raw=... h_tgt=... h_corr=... corr=...
TASK3_BD stop: reason=line/force ...
```

判断：

- `reason=line`：BD 扫到 D 点线，进入 DA。
- `reason=force`：BD 没扫到 D 点线，按编码器距离兜底停车，说明角度、寻线距离或红外判定仍要调。

### 4.2 D 点入线判定目前偏宽

当前 `TASK2_CD_STOP_CENTER_MASK` 为 `0xFF`，BD 也复用这套宽松判定。优点是容易扫到线，缺点是边缘探头碰到线也可能进入 DA。

如果后续出现“还没到 D 点就进入 DA”，建议把 BD 的入线判定单独收紧，例如要求中间探头或 `active_count>=2`。不要直接影响任务一已经稳定的停车逻辑。

### 4.3 任务三弧线丢线仍需继续观察

任务三 CB/DA 弧线是专用逻辑，关键字段：

- `valid`：红外当前是否有效看到线。
- `lost_cnt`：连续丢线次数。
- `entry`：是否还在入弯强化区。
- `mode`：当前弧线控制模式。
- `turn/L/R`：转向量和左右轮 PWM。

如果 CB 或 DA 弧线中段丢线，优先调 `TASK3_ARC_*` 参数，不要改任务二弧线参数。

## 6. 后续开发建议

1. 先围绕任务三 `TASK3_BD_RELATIVE_TURN_CDEG` 小步调参。
2. 若 BD 仍到不了 D 点，先看 `h_raw/h_tgt` 是否转到目标，再判断是角度问题还是红外入线问题。
3. 若 BD 角度已经正确但仍扫不到线，调整 BD 的寻线开始距离和摆头强度。
4. 若 DA 末端停车滞后，继续收紧最终 A 点出线判定，保证声光只由红外出线事件触发。
5. 任务三稳定后再开始任务四，不要同时大改多个任务。
