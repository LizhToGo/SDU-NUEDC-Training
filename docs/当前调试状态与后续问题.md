# 当前调试状态与后续问题

日期：2026-06-08

本文记录当前 `main` 分支的实车固件状态、关键参数和后续调试重点。这个版本相对 6 月初留档版改动很大，旧文档中“任务三/四使用 B_exit/A_exit 加相对转角”的说法已经不再符合当前代码；当前任务三和任务四使用固定绝对航向目标，并依赖 Task10 建立的 AB 零航向基准。

## 1. 当前可用状态

- 正式任务调度已启用：`ENABLE_CONTEST_TASKS=1`。
- UART0 支持 ASCII 命令 `00..07/10/11`，也支持原始 HEX 字节 `0x01..0x07/0x10/0x11`。
- 运行中发送 `00` 或原始 `0x00` 会请求停车。
- 实体任务按键：
  - `A26 / PA26`：任务一。
  - `A24 / PA24`：任务二。
  - `B24 / PB24`：任务三。
  - `A22 / PA22`：任务四。

| 命令 | 当前用途 | 代码入口 |
|---|---|---|
| `01` | 任务一 A -> B 直线 | `run_task1_ab()` |
| `02` | 任务二 A -> B -> C -> D -> A | `run_task2_abcd()` |
| `03` | 任务三 A -> C -> B -> D -> A | `run_task3_acbda()` |
| `04` | 任务四，按任务三路线连续 4 圈 | `run_task4_four_laps()` |
| `05` | PID 轮速/编码器/PWM 测试 | `run_motor_pid_stream()` |

| `07` | PD 轮速/编码器/PWM 测试 | `run_motor_pd_stream()` |
| `10` | Task10，AB 方向零航向标定 | `run_task10_ab_zero_test()` |
| `11` | Task11，全程黑线经验采集 | `run_task11_ir_map_test()` |

## 2. 当前关键参数

### 2.1 基础直线

```c
#define STRAIGHT_B_BASE_PWM (626)
#define STRAIGHT_A_BASE_PWM (633)
#define STRAIGHT_MIN_PWM    (0)
#define STRAIGHT_MAX_PWM    (870)

#define TASK1_RAMP_B_START_PWM (560)
#define TASK1_RAMP_A_START_PWM (600)
#define TASK1_START_RAMP_MS    (400)

#define TASK1_B_LINE_ARM_COUNT (6000)
#define TASK1_FORCE_STOP_COUNT (9500)
```

任务一和任务二 AB 使用 `run_task5_straight_to_line_segment()`，核心控制来自 `app_straight.h` 的差速控制模块。

### 2.2 任务三/四绝对航向

```c
#define TASK3_AC_HEADING_TARGET_CDEG  (-3660)
#define TASK3_BD_HEADING_TARGET_CDEG  (-13920)
#define TASK4_AC_HEADING_TARGET_CDEG  TASK3_AC_HEADING_TARGET_CDEG
#define TASK4_BD_HEADING_TARGET_CDEG  TASK3_BD_HEADING_TARGET_CDEG
```

单位为 0.01 度：

- `-3660` 表示 AC 直线目标为 -36.60 度。
- `-13920` 表示 BD 直线目标为 -139.20 度。

`TASK3_BD_RELATIVE_TURN_CDEG` 当前仍保留在配置文件中，值为 `6600`，但当前任务三/四的实际分段逻辑打印 `mode=absolute`，不再使用它来计算 BD 或下一圈 AC 目标角。

跑任务三/四前推荐流程：

1. 车头朝 AB 方向放置。
2. UART0 发送 `10`，确认 `TASK10 zero_ab: ok=1 nav=1 rel=0`。
3. 把车放到任务起点，任务三/四按 A 点车头朝 C 启动。

### 2.3 Task11 风格弧线控制

```c
#define TASK11_LINE_BASE_PWM             (560)
#define TASK11_ARC_BASE_PWM              (540)
#define TASK11_CB_ARC_ENTRY_TARGET_DIFF  (-44)
#define TASK11_CB_ARC_CRUISE_TARGET_DIFF (-44)
#define TASK11_DA_ARC_ENTRY_TARGET_DIFF  (42)
```

任务二、任务三和任务四的弧线段已经大量使用 Task11 风格控制：差速目标提供基础转弯趋势，红外误差给 `line_turn`，JY62 弧线模型给 `nav_turn`，最终叠加成 `turn`。

## 3. 当前任务逻辑摘要

### 3.1 任务一

任务一现在走 `run_task5_straight_to_line_segment("TASK1_AB", ...)`：

1. 启动声光。
2. JY62 当前航向可置零。
3. `straight_drive_update()` 根据左右轮速度差、累计距离差和前馈/反馈修正左右 PWM。
4. JY62 航向修正通过 `task5_yaw_correction()` 叠加。
5. 到 `TASK1_B_LINE_ARM_COUNT` 后允许红外触发 B 点停车。

看直线偏航时，重点看 `yaw/ycorr/B_total/A_total/d_err/ff/fb/corr/B_pwm/A_pwm`。

### 3.2 任务二

任务二是调试组合版本：

- AB：任务一风格直线。
- BC：`run_task2_bc_task11_arc_debug()`。
- CD：`run_task2_cd_exit_angle_straight()`，固定目标角来自 `TASK2_CD_STRAIGHT_TARGET_CDEG`。
- DA：`run_task2_da_task11_arc()`。

任务二保留 RAM 日志结构，主要用于分析弧线、CD 出线和固定目标角。

### 3.3 任务三

任务三当前路线：

```text
TASK3_AC -> TASK3_CB -> TASK3_BD -> TASK3_DA
```

关键点：

- AC 使用固定目标角 `TASK3_AC_HEADING_TARGET_CDEG`。
- CB 使用 `run_task3_task11_arc_line_follow_segment()`，左弧线。
- BD 使用固定目标角 `TASK3_BD_HEADING_TARGET_CDEG`，日志会打印 `mode=absolute`。
- DA 使用同一套 Task11 风格弧线，右弧线，最终 A 点停车。

### 3.4 任务四

任务四连续跑 4 圈，每圈使用：

```text
AC 固定目标角 -> CB 左弧线 -> BD 固定目标角 -> DA 右弧线
```

前三圈 DA 出 A 点后不结束，只触发声光并进入下一圈；第四圈 A 点后停车。

当前 `TASK4_A target seed` 日志仍会打印 `A_exit`，但下一圈目标固定为 `TASK4_AC_HEADING_TARGET_CDEG`，不是 `A_exit + 相对转角`。

### 3.5 任务十/十一

Task10：

- 车头朝 AB 方向摆放。
- 发送 `10`。
- 正常输出：`TASK10 zero_ab: ok=1 nav=1 rel=0 ...`。

Task11：

- 车头从 A 朝 C 放置。
- 场地需要 AC 和 BD 也贴黑线。
- 按 `AC -> CB -> BD -> DA` 跑 5 圈。
- 用于采集任务三/四可复用的距离、航向、转向和点位识别经验。
- 目前默认关闭大体量日志，打开 RAM 日志后会在任务结束输出 `TASK11_RAM_BEGIN` 到 `TASK11_RAM_END`。

## 5. 当前已知问题

### 5.1 文档和历史参数容易混淆

旧文档大量描述 `B_exit + 6700` 和 `A_exit - 6700`。当前代码已经改为绝对目标角，调车时应以 `TASK3_AC_HEADING_TARGET_CDEG` 和 `TASK3_BD_HEADING_TARGET_CDEG` 为准。

### 5.2 Task11 日志默认关闭

如果需要继续用 Task11 采集经验数据，必须确认 `TASK11_RAM_LOG_ENABLE` 或 `TASK11_UART_LOG_ENABLE` 已打开。否则任务可以跑，但不会输出完整 `TASK11_SUM/WIN/EVT` 数据。

### 5.3 经验数据文件已经被加入 ignore

`.gitignore` 已新增：

```text
/task11_experience_*.csv
/task11_experience_summary.txt
/data/task11_experience_*.csv
/data/task11_experience_summary.txt
```

注意：如果这些文件已经被 Git 跟踪，`.gitignore` 不会自动把它们从历史中移除。若后续决定从仓库删除，需要单独执行 `git rm --cached ...`。

### 5.4 任务四仍需实车验证连续稳定性

重点看：

- 每圈 AC 是否能按固定目标角稳定到 C。
- CB/DA 弧线是否在中后段丢线。
- BD 固定目标角是否稳定到 D。
- 第四圈 DA 是否由红外点位触发，而不是距离兜底。

## 7. 后续调试建议

1. 先确认任务一当前 `626/633` 直线参数仍然稳定。
2. 再用任务三验证 AC 固定目标角和 BD 固定目标角。
3. 如果任务三能完整跑，再跑任务四观察每圈误差是否积累。
4. 若任务四中某圈 AC/BD 偏离，优先调整绝对航向目标，而不是旧的相对转角。
5. 若弧线段丢线，优先看 `TASK3_CB/TASK3_DA` 或 `TASK4_Lx_CB/DA` 的 `line_turn/nav_turn/tdiff/pwm`。
