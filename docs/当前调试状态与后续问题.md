# 当前调试状态与后续问题

日期：2026-06-01

本文记录准备 push 前这一版固件的实车状态、关键参数和后续调试重点。当前目标是保留一份可编译、可复现实测状态的基线，方便后续继续调任务四四圈稳定性。

## 1. 当前可用状态

- 当前正式任务调度已启用：`ENABLE_CONTEST_TASKS=1`。
- 空闲等待时不再持续刷 `JY62 mode=idle`；收到任务命令后开始输出对应任务数据。
- UART0 命令：
  - `01`：任务一 A -> B 直线。
  - `02`：任务二 A -> B -> C -> D -> A。
  - `03`：任务三 A -> C -> B -> D -> A。
  - `04`：任务四，按任务三路线连续 4 圈。
  - `05`：左右轮编码器/PWM 测试。
  - `00`：强制停车。
- 实体任务按键：
  - `A26 / PA26`：任务一。
  - `A24 / PA24`：任务二。
  - `B24 / PB24`：任务三。
  - `A22 / PA22`：任务四。
- 任务一当前直线稳定，可作为基础直线留档。
- 任务二可运行，仍需继续看弧线入口和出线姿态。
- 任务三已实现完整路线，`TASK3_BD_RELATIVE_TURN_CDEG=6700` 是当前实测可用值。
- 任务四已实现基础四圈逻辑，目前属于勉强可跑的留档版本，后续继续优化连续换圈稳定性和用时。

## 3. 任务三当前逻辑

任务三入口：`B24` 或 UART0 `03`。

流程：

1. `TASK3_AC`：A -> C 直线。启动时 JY62 置零，直线控制逻辑和任务一相同，接近 C 点后开启红外寻线。
2. C 点：AC 直线扫到入线后触发声光，进入 CB 弧线。
3. `TASK3_CB`：任务三专用红外弧线。入弯初期给更强的转向辅助，随后由红外误差闭环循迹。
4. B 点：CB 段中，当航向进度和 B 点出线条件成立，并且红外判定出线时，`reason=arc_done`。此时触发声光，不刹车，直接进入 BD。
5. `TASK3_BD`：B -> D 直线。目标航向不是固定角，而是 `B_exit + TASK3_BD_RELATIVE_TURN_CDEG`，当前相对左转 67 度。
6. D 点：BD 直线扫到线后触发声光，进入 DA 弧线。
7. `TASK3_DA`：任务三专用红外弧线。最终 A 点由红外出线触发停车和结束声光。

任务三详细字段见 [任务三调试说明.md](任务三调试说明.md)。

## 4. 任务四当前逻辑

任务四入口：`A22` 或 UART0 `04`。

任务四按任务三路线连续跑 4 圈，当前作为基础可跑版本留档：

1. 第一圈从 A 点启动，车头朝 C，JY62 当前方向置零。
2. 每圈 AC、CB、BD、DA 的控制逻辑与任务三一致。
3. B 点出线后使用 `B_exit + TASK3_BD_RELATIVE_TURN_CDEG`，当前相对左转 `+6700`。
4. 前三圈 A 点出线后不刹车，触发声光后进入下一圈。
5. A 点换圈使用 `A_exit + TASK4_AC_RELATIVE_TURN_CDEG`，当前相对右转 `-6700`，与 B 点等大反向。
6. 第四圈 DA 到 A 点后刹车并触发结束声光。

串口会按圈数打印类似 `TASK4_L1_AC`、`TASK4_L2_CB`、`TASK4_L3_BD`、`TASK4_L4_DA` 的 tag，方便区分每一圈。

更详细的任务四字段和调参方法见 [任务四调试说明.md](任务四调试说明.md)。

## 5. 当前已知问题

### 5.1 任务四仍需实车验证

重点看：

- 前三圈 A 点出线后是否能不停车顺利进入下一圈 AC。
- `TASK4_A target seed` 中的 `A_exit` 和 `next_target` 是否合理。
- 下一圈 `TASK4_L*_AC` 是否能扫到 C 点入线。

看日志时重点关注：

```text
TASK4_A target seed: lap=... A_exit=... rel_turn=-6700 next_target=...
TASK4_L2_AC t=... h_raw=... h_tgt=... h_corr=...
TASK4_L2_AC stop: reason=line/force ...
```

判断：

- `reason=line`：下一圈 AC 扫到 C 点线，进入 CB。
- `reason=force`：下一圈 AC 没扫到 C 点线，优先看 A 点换圈角度和 AC 寻线距离。

### 5.2 D 点入线判定目前偏宽

当前 `TASK2_CD_STOP_CENTER_MASK` 为 `0xFF`，BD 也复用这套宽松判定。优点是容易扫到线，缺点是边缘探头碰到线也可能进入 DA。

如果后续出现“还没到 D 点就进入 DA”，建议把 BD 的入线判定单独收紧，例如要求中间探头或 `active_count>=2`。不要直接影响任务一已经稳定的停车逻辑。

### 5.3 任务三/四弧线丢线仍需继续观察

任务三 CB/DA 弧线是专用逻辑，关键字段：

- `valid`：红外当前是否有效看到线。
- `lost_cnt`：连续丢线次数。
- `entry`：是否还在入弯强化区。
- `mode`：当前弧线控制模式。
- `turn/L/R`：转向量和左右轮 PWM。

如果 CB 或 DA 弧线中段丢线，优先调 `TASK3_ARC_*` 参数，不要改任务二弧线参数。

## 7. 后续开发建议

1. 先用任务三确认 `6700` 仍然稳定。
2. 再跑任务四，优先看第一圈 A 点换圈后的 `TASK4_L2_AC`。
3. 如果第二圈 AC 到不了 C，先小步调 `TASK4_AC_RELATIVE_TURN_CDEG`，不要改任务三 B 点 `6700`。
4. 如果换圈角度正确但扫不到线，再调任务四 AC 寻线距离和摆头强度。
5. 第四圈停车如果滞后，继续收紧 DA 最终 A 点出线判定。
