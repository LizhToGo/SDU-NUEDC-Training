# 当前调试状态与后续问题

日期：2026-06-15（代码结构重构后）

本文记录当前主分支的实车固件状态、关键参数和后续调试重点。当前任务三/四已经不再使用旧 Task3/Task4 专用流程，也没有 UART `11` 任务入口；正式竞速流程统一由 `run_race_laps()` 执行。

## 1. 当前可用状态

- 正式任务调度已启用：`ENABLE_CONTEST_TASKS=1`。
- UART0 支持 ASCII 命令 `00..07/10`，也支持原始 HEX 字节 `0x01..0x07/0x10/0x00`。
- 运行中发送 `00` 或原始 `0x00` 会请求停车。
- 实体任务按键：
  - `A26 / PA26`：任务一。
  - `A24 / PA24`：任务二。
  - `B24 / PB24`：任务三。
  - `A22 / PA22`：任务四。

| 命令 | 当前用途 | 代码入口 |
|---|---|---|
| `01` | 任务一 A -> B 直线 | `run_task1_ab()` |
| `02` | 任务二 A -> B -> C -> D -> A | `run_task2_abcd()` |
| `03` | 任务三 A -> C -> B -> D -> A | `run_race_laps(1U)` |
| `04` | 任务四，按任务三路线连续 4 圈 | `run_race_laps(4U)` |
| `05` | PID 轮速/编码器/PWM 测试 | `run_motor_pid_stream()` |

| `07` | PD 轮速/编码器/PWM 测试 | `run_motor_pd_stream()` |
| `10` | AB 方向零航向标定+持续航向监控 | `run_task10_ab_zero_test()` |

## 3. 当前任务逻辑摘要

### 3.1 任务一

任务一走 `run_straight_to_line_segment()`：

1. 启动声光。
2. JY62 当前航向可置零。
3. `straight_drive_update()` 根据左右轮速度差、累计距离差和前馈/反馈修正左右 PWM。
4. 航向修正叠加到最终 `corr`。
5. 到 `TASK1_B_LINE_ARM_COUNT` 后允许红外触发 B 点停车。

### 3.2 任务二

任务二是调试组合版本：

- AB：任务一风格直线。
- BC：`run_task2_bc_race_arc_debug()`。
- CD：`run_task2_cd_exit_angle_straight()`。
- DA：`run_task2_da_race_arc()`。

任务二保留 RAM 日志结构，主要用于分析弧线、CD 出线和固定目标角。

### 3.3 任务三

任务三当前走 `run_race_laps(1U)`，即竞速单圈模式。

```text
RACE_AC -> RACE_CB -> RACE_BD -> RACE_DA
```

关键点：

- AC 使用 `TASK11_AC_HEADING_TARGET_CDEG=-50`。
- CB 使用竞速弧线控制：差速目标 + 红外 PD + 可选 JY62 弧线航向修正。
- B 点出口使用 `race_gyro_turn_to_yaw()`，目标为 `TASK11_BD_HEADING_TARGET_CDEG`。
- BD 使用 `TASK11_BD_HEADING_TARGET_CDEG=-10638`。
- DA 使用竞速弧线控制，A 点出口同样可用 `race_gyro_turn_to_yaw()`。

### 3.4 任务四

任务四走 `run_race_laps(4U)`，连续 4 圈竞速路径。前三圈 DA 出 A 点后进入下一圈；第四圈 A 点后停车。

### 3.5 任务十

Task10 用于建立 AB 零航向：

```text
TASK UART command accepted: id=10
JY62 mode=TASK10_AB_ZERO ...
TASK10 zero_ab: ok=1 nav=1 rel=0 ...
```

## 5. 当前已知问题

### 5.1 命名处于过渡状态

实现函数和日志已经是 `race_*` / `RACE_*`，但配置宏仍为 `TASK11_*`。调参时不要误以为这些宏无效；它们仍是当前竞速路径的实际参数。

### 5.2 日志脚本保留历史文件名

`tools/task11_log_to_csv.py` 仍保留历史 Task11 文件名和默认输出路径，但用途已经扩展到当前 `RACE_*` RAM 日志。若后续彻底清理命名，需要同步兼容旧日志和新日志。

### 5.3 任务四仍需实车验证连续稳定性

重点看：

- 每圈 AC 是否稳定到 C。
- CB/DA 弧线点位是否正确触发。
- B/A 点 `race_gyro_turn_to_yaw()` 是否按绝对航向稳定停止。
- BD 是否稳定到 D。
- 第四圈 DA 是否由红外点位触发，而不是距离兜底。
