# 当前调试状态与后续问题

日期：2026-06-08（commit 0199d70 更新）

**commit 0199d70 重大变更**：任务三/四入口改走 `run_task11_ir_map_test_laps()`，B/A 点出口转向从红外 `sensor_fast_turn` 改为陀螞仪 `gyro_turn_to_yaw`，Task11 使用独立航向目标。

本文记录当前 `main` 分支的实车固件状态、关键参数和后续调试重点。这个版本相对 6 月初留档版改动很大，旧文档中“任务三/四使用 B_exit/A_exit 加相对转角”的说法已经不再符合当前代码；当前任务三和任务四使用固定绝对航向目标，并依赖 Task10 建立的 AB 零航向基准。

- 正式任务调度已启用：`ENABLE_CONTEST_TASKS=1`。

- 运行中发送 `00` 或原始 `0x00` 会请求停车。
- 实体任务按键：
  - `A26 / PA26`：任务一。
  - `A24 / PA24`：任务二。

## 2. 当前关键参数

### 2.1 基础直线

```c
#define STRAIGHT_B_BASE_PWM (628)
#define STRAIGHT_A_BASE_PWM (633)
#define STRAIGHT_MIN_PWM    (0)
#define STRAIGHT_MAX_PWM    (870)

#define TASK1_RAMP_B_START_PWM (560)
#define TASK1_RAMP_A_START_PWM (600)
#define TASK1_START_RAMP_MS    (400)

#define TASK1_B_LINE_ARM_COUNT (6000)
#define TASK1_FORCE_STOP_COUNT (9500)
```

任务一和任务二 AB 使用 `run_straight_to_line_segment()`，核心控制来自 `app_straight.h` 的差速控制模块。

### 2.2 任务三/四绝对航向

```c
#define TASK3_AC_HEADING_TARGET_CDEG  (-3660)
#define TASK3_BD_HEADING_TARGET_CDEG  (-13920)
#define TASK4_AC_HEADING_TARGET_CDEG  TASK3_AC_HEADING_TARGET_CDEG
#define TASK4_BD_HEADING_TARGET_CDEG  TASK3_BD_HEADING_TARGET_CDEG
```

单位为 0.01 度：

- `-3660` 表示 AC 直线目标为 -36.60 度。
- `-13920` 表示 BD 直线目标为 -139.20 度。

`TASK3_BD_RELATIVE_TURN_CDEG` 当前仍保留在配置文件中，值为 `6600`，但当前任务三/四的实际分段逻辑打印 `mode=absolute`，不再使用它来计算 BD 或下一圈 AC 目标角。

跑任务三/四前推荐流程：

1. 车头朝 AB 方向放置。
2. UART0 发送 `10`，确认 `TASK10 zero_ab: ok=1 nav=1 rel=0`。
3. 把车放到任务起点，任务三/四按 A 点车头朝 C 启动。

### 2.3 Task11 风格弧线控制

```c
#define TASK11_LINE_BASE_PWM             (560)
#define TASK11_ARC_BASE_PWM              (540)
#define TASK11_CB_ARC_ENTRY_TARGET_DIFF  (-48)
#define TASK11_CB_ARC_CRUISE_TARGET_DIFF (-48)
#define TASK11_DA_ARC_ENTRY_TARGET_DIFF  (46)
#define TASK11_DA_ARC_CRUISE_TARGET_DIFF (46)

#define TASK11_ARC_YAW_NAV_ENABLE        (0)
#define TASK11_ARC_YAW_CORR_DIVISOR      (260)
#define TASK11_ARC_YAW_CORR_MAX          (45)
#define TASK11_ARC_POINT_YAW_ARM_CDEG    (14000)
```

任务二、任务三和任务四的弧线段已经大量使用 Task11 风格控制：差速目标提供基础转弯趋势，红外误差给 `line_turn`，JY62 弧线模型给 `nav_turn`，最终叠加成 `turn`。

### 2.4 Task11 采集参数

#define TASK11_TARGET_LAPS               (5)

当前默认：

```c

#define TASK11_RAM_LOG_ENABLE            (0)

```

也就是说，当前版本默认不输出大体量 Task11 日志。需要采集数据时，应在 `app_config.h` 中打开 `TASK11_RAM_LOG_ENABLE` 或实时日志开关，再用 `tools/task11_log_to_csv.py` 整理。

## 3. 当前任务逻辑摘要

### 3.1 任务一

任务一现在走 `run_straight_to_line_segment("TASK1_AB", ...)`：

1. 启动声光。
2. JY62 当前航向可置零。
3. `straight_drive_update()` 根据左右轮速度差、累计距离差和前馈/反馈修正左右 PWM。
4. JY62 航向修正通过 `task5_yaw_correction()` 叠加。
5. 到 `TASK1_B_LINE_ARM_COUNT` 后允许红外触发 B 点停车。

看直线偏航时，重点看 `yaw/ycorr/B_total/A_total/d_err/ff/fb/corr/B_pwm/A_pwm`。

### 3.2 任务二

任务二是调试组合版本：

- AB：任务一风格直线。
- BC：`run_task2_bc_task11_arc_debug()`。
- CD：`run_task2_cd_exit_angle_straight()`，固定目标角来自 `TASK2_CD_STRAIGHT_TARGET_CDEG`。
- DA：`run_task2_da_task11_arc()`。

任务二保留 RAM 日志结构，主要用于分析弧线、CD 出线和固定目标角。

### 3.3 任务三

任务三当前走 `run_task11_ir_map_test_laps(1U)`，即 Task11 单圈模式。

```text
TASK11_AC -> TASK11_CB -> TASK11_BD -> TASK11_DA
```

关键点：

- AC 使用 Task11 专用目标角 `TASK11_AC_HEADING_TARGET_CDEG=-50`。
- CB 使用 Task11 弧线控制（差速+红外），弧线点位判定改为 `line_lost_seen`。
- B 点出口转向从红外 `sensor_fast_turn` 改为陀螞仪 `gyro_turn_to_yaw`，使用 `TASK11_B_EXIT_TARGET_CDEG`。
- BD 使用 Task11 专用目标角 `TASK11_BD_HEADING_TARGET_CDEG=-10638`。
- DA 使用 Task11 弧线控制，A 点出口转向同样改为 `gyro_turn_to_yaw`。

### 3.4 任务四

任务四走 `run_task11_ir_map_test_laps(4U)`，连续 4 圈 Task11 路径。

每圈使用：

```text
AC(Task11目标角) -> CB(弧线) -> BD(Task11目标角) -> DA(弧线)
```

前三圈 DA 出 A 点后不结束，只触发声光并进入下一圈；第四圈 A 点后停车。

B/A 点出口转向均使用 `gyro_turn_to_yaw`，按绝对航向停止。

### 3.5 任务十/十一

Task10：

- 车头朝 AB 方向摆放。
- 发送 `10`。
- 正常输出：`TASK10 zero_ab: ok=1 nav=1 rel=0 ...`。

Task11：

- 车头从 A 朝 C 放置。
- 场地需要 AC 和 BD 也贴黑线。
- 按 `AC -> CB -> BD -> DA` 跑 5 圈（可通过 `run_task11_ir_map_test_laps(n)` 指定圈数）。
- 当前 Task11 使用独立航向目标 `TASK11_AC_HEADING_TARGET_CDEG=-50`、`TASK11_BD_HEADING_TARGET_CDEG=-10638`。
- B/A 点出口转向使用 `gyro_turn_to_yaw`（陀螞仪绝对航向转向）。
- 用于采集任务三/四可复用的距离、航向、转向和点位识别经验。
- 目前默认关闭大体量日志，打开 RAM 日志后会在任务结束输出 `TASK11_RAM_BEGIN` 到 `TASK11_RAM_END`。

## 5. 当前已知问题

### 5.1 文档和历史参数容易混淆

旧文档大量描述 `B_exit + 6700` 和 `A_exit - 6700`。当前代码已经改为绝对目标角，调车时应以 `TASK3_AC_HEADING_TARGET_CDEG` 和 `TASK3_BD_HEADING_TARGET_CDEG` 为准。

### 5.2 Task11 日志默认关闭

如果需要继续用 Task11 采集经验数据，必须确认 `TASK11_RAM_LOG_ENABLE` 或 `TASK11_UART_LOG_ENABLE` 已打开。否则任务可以跑，但不会输出完整 `TASK11_SUM/WIN/EVT` 数据。

### 5.3 经验数据文件已经被加入 ignore

`.gitignore` 已新增：

```text
/task11_experience_*.csv
/task11_experience_summary.txt
/data/task11_experience_*.csv
/data/task11_experience_summary.txt
```

注意：如果这些文件已经被 Git 跟踪，`.gitignore` 不会自动把它们从历史中移除。若后续决定从仓库删除，需要单独执行 `git rm --cached ...`。

### 5.4 任务四仍需实车验证连续稳定性

重点看：

- 每圈 AC 是否能按 Task11 专用目标角稳定到 C。
- CB/DA 弧线点位是否由 `line_lost_seen` 正确触发。
- B/A 点 `gyro_turn_to_yaw` 是否按绝对航向稳定停止。
- BD Task11 专用目标角是否稳定到 D。
- 第四圈 DA 是否由红外点位触发，而不是距离兜底。

## 7. 后续调试建议

1. 先确认任务一当前 `628/633` 直线参数仍然稳定。
2. 用 Task10 持续监控模式确认 JY62 零航向漂移在可接受范围内。
3. 再用任务三（`run_task11_ir_map_test_laps(1U)`）验证 Task11 专用航向目标和 `gyro_turn_to_yaw`。
4. 如果任务三能完整跑，再跑任务四观察每圈误差是否积累。
5. 若任务四中某圈 AC/BD 偏离，优先调整 `TASK11_AC/BD_HEADING_TARGET_CDEG`。
6. 若弧线段丢线，看 `TASK11_CB/DA` 的 `line_turn/nav_turn/tdiff/pwm`。
7. 若 `gyro_turn_to_yaw` 过冲或不足，调整 `TASK11_TURN_YAW_STOP_TOL_CDEG` 和 `TASK11_TURN_YAW_SLOW_ZONE_CDEG`。
